## 我的目标

- 身份目标：Java 后端初级 -> Python AI 应用开发工程师
- 能力目标：能独立完成 AI 模块设计、Python 接口封装、RAG/Agent 基础项目落地，并提供给 Java 后端调用
- 项目目标：做出 2-3 个可展示、可部署、可讲清楚的 AI 工程项目

## AI 当老师的工作方式

### 教学原则

1. 先讲结论，再讲原理，再讲代码。
2. 每次只学一个小块，必须有输出。
3. 不懂就拆到最小概念，不跳步。
4. 学完必须落到代码、接口、项目或文档。
5. 每章结束必须复盘：能否讲清楚、能否写进简历、能否独立复现。

### 你要我重点盯的事

- 哪些内容可以写进简历
- 哪些内容还只是“会了”，但不够企业级
- 哪些地方可以优化成工程化、可部署、可维护
- 哪些点适合面试表达
- 哪些项目能真正帮助拿 offer

### 我的反馈标准

- 基础层：会不会、懂不懂
- 实战层：能不能写、能不能跑
- 工程层：稳不稳定、好不好维护
- 面试层：能不能讲清楚、能不能体现价值

## 简历写法规则

### 可以写进简历的内容

- 独立完成了可运行项目
- 做了 FastAPI / Flask / Django 接口封装
- 接入了大模型 API
- 做了 Prompt 设计、多轮对话、流式输出
- 实现了 RAG 检索流程
- 做了工具调用 / Function Calling / 简单 Agent
- 使用 Redis、MySQL、Docker、Nginx、Celery 等做工程化落地
- 项目有部署、有日志、有异常处理、有文档

### 暂时不要写得太满的内容

- 只看过文档但没真正跑通
- 只会调用接口，但说不清原理
- 只会 demo，不会封装成服务
- 只会本地运行，不会部署
- 只会“知道”，没有产出

### 简历表达模板

- “基于 FastAPI 封装 AI 推理服务，支持流式输出与异常处理”
- “实现 RAG 问答模块，完成文档切分、向量化、检索和生成”
- “设计 Prompt 模板和多轮对话上下文管理机制”
- “使用 Redis/Celery/Docker 完成 AI 服务工程化部署”

## 企业级优化清单

- 接口规范统一
- 参数校验完整
- 错误码清晰
- 日志可追踪
- 配置外置化
- 支持流式输出
- 支持超时和重试
- 支持缓存
- 支持异步任务
- 支持 Docker 部署
- 支持基础测试
- 有 README 和接口说明
