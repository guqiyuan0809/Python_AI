# ReAct 模式详解

> ReAct = Reasoning + Acting（推理 + 行动）
> 这是目前最主流的 Agent 模式，边想边做，做完再想。

---

## 一、ReAct 是什么

一句话：AI 每一步都先说"我在想什么"，再说"我要做什么"，做完看结果，再想下一步。

你在 Agent Skills Demo 2 里写的循环就是 ReAct 的简化版，只不过没有显式的"思考"步骤。

### 标准 ReAct 流程

```
用户：帮我查北京和上海的天气，告诉我哪个更热

Thought 1: 用户想比较两个城市的温度，我需要先查北京的天气
Action 1:  get_weather("北京")
Observation 1: 晴天，35°C

Thought 2: 北京 35°C，现在查上海的天气
Action 2:  get_weather("上海")
Observation 2: 多云，22°C

Thought 3: 北京 35°C，上海 22°C，北京更热，我可以回答了
Answer:    北京（35°C）比上海（22°C）更热，高了 13 度。
```

三个关键词：
- Thought（思考）：AI 说出自己的推理过程
- Action（行动）：调用工具
- Observation（观察）：看工具返回的结果

---

## 二、ReAct vs 你之前写的 Agent 循环

### Agent Skills Demo 2 的循环（简化版）

```python
while True:
    response = call_ai(messages, tools)
    if response.tool_calls:
        execute_tools()      # 直接执行，没有显式思考
    else:
        return response      # 结束
```

AI 内部其实在思考，但你看不到思考过程。

### ReAct 模式（完整版）

```python
while True:
    response = call_ai(messages, tools)
    
    # AI 会先输出 Thought（思考过程）
    if response.content:
        print(f"Thought: {response.content}")  # 能看到 AI 在想什么
    
    if response.tool_calls:
        result = execute_tools()               # Action（行动）
        print(f"Observation: {result}")         # Observation（观察）
    else:
        return response.content                # 最终回答
```

区别：ReAct 让 AI 的思考过程可见，方便调试和理解。

## 三、为什么 ReAct 有效

### 1. 思维链（Chain of Thought）效应

让 AI "说出思考过程" 能提高推理准确度。这和提示词工程里的 "让 AI 一步步思考" 是同一个原理。

```
不让 AI 思考：
  用户：123 * 456 + 789 = ?
  AI：57057（可能算错）

让 AI 思考：
  用户：123 * 456 + 789 = ?
  AI：先算 123 * 456 = 56088，再加 789 = 56877（更准确）
```

### 2. 自我纠错

AI 看到工具返回的结果后，可以判断结果是否合理，不合理就换个方法：

```
Thought: 我需要查用户的订单信息
Action: query_database("SELECT * FROM orders WHERE user_id = 123")
Observation: 查询错误，表名不对

Thought: 表名可能不是 orders，让我查一下有哪些表
Action: query_database("SHOW TABLES")
Observation: users, order_records, products

Thought: 原来表名是 order_records，重新查询
Action: query_database("SELECT * FROM order_records WHERE user_id = 123")
Observation: [订单数据...]
```

如果没有 ReAct 的观察-思考循环，AI 第一次出错就卡住了。

### 3. 可解释性

你能看到 AI 每一步在想什么、为什么做这个决定，出了问题容易排查。

## 四、ReAct 的实现方式

### 方式一：通过 System Prompt 引导（简单）

```python
system_prompt = """你是一个助手，请按照以下格式回答：

Thought: [你的思考过程]
Action: [要调用的工具和参数]
Observation: [工具返回的结果]
... （重复直到任务完成）
Answer: [最终回答]

每一步都要先 Thought，再 Action。"""
```

这种方式依赖 AI 自觉遵守格式，不够可靠。

### 方式二：通过代码控制（推荐）

```python
def react_agent(user_message, tools, max_steps=10):
    messages = [
        {"role": "system", "content": "你是一个助手。每次回答前先思考，说出你的推理过程。"},
        {"role": "user", "content": user_message}
    ]
    
    for step in range(max_steps):
        response = client.chat.completions.create(
            model=MODEL,
            messages=messages,
            tools=tools,
        )
        
        message = response.choices[0].message
        
        # Thought: AI 的思考（content 部分）
        if message.content:
            print(f"Thought: {message.content}")
        
        # Action: 调用工具
        if message.tool_calls:
            messages.append(message)
            for tool_call in message.tool_calls:
                result = execute_tool(tool_call)
                print(f"Action: {tool_call.function.name}({tool_call.function.arguments})")
                print(f"Observation: {result}")
                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": result
                })
        else:
            # 没有工具调用，任务完成
            print(f"Answer: {message.content}")
            return message.content
    
    return "达到最大步数，任务未完成"
```

代码控制循环，AI 负责思考和决策，这是最可靠的方式。

## 五、ReAct 的完整执行示例

```
用户：帮我分析北京和上海哪个城市更适合旅游，考虑天气和人口

━━━ Step 1 ━━━
Thought: 用户想比较两个城市，需要天气和人口数据。先查北京的天气。
Action:  get_weather("北京")
Observation: 晴天，35°C

━━━ Step 2 ━━━
Thought: 北京 35°C，有点热。接下来查上海的天气。
Action:  get_weather("上海")
Observation: 多云，22°C

━━━ Step 3 ━━━
Thought: 天气数据有了。北京 35°C 偏热，上海 22°C 舒适。现在查人口。
Action:  get_population("北京")
Observation: 2170 万人

━━━ Step 4 ━━━
Thought: 北京 2170 万人，再查上海。
Action:  get_population("上海")
Observation: 2490 万人

━━━ Step 5 ━━━
Thought: 数据齐了。天气方面上海 22°C 更舒适，人口方面两个城市都很拥挤，
        但上海稍多。综合来看上海天气更适合旅游。可以给出最终回答了。
Answer:  从旅游角度看，上海目前更适合：
        - 天气：上海 22°C 多云，体感舒适；北京 35°C 偏热
        - 人口：两城都是超大城市，北京 2170 万，上海 2490 万
        建议选择上海，天气更宜人。
```

注意 AI 在每一步都说了"我在想什么"，这就是 ReAct 的核心。

## 六、ReAct 的优缺点

### 优点

| 优点 | 说明 |
|------|------|
| 可解释 | 能看到 AI 每步的思考过程 |
| 能纠错 | 看到错误结果后可以调整策略 |
| 灵活 | 不需要预先规划所有步骤 |
| 简单 | 实现起来就是循环（for/while）+ Function Calling |

### 缺点

| 缺点 | 说明 |
|------|------|
| 可能绕圈 | AI 可能反复做同样的事情 |
| 成本高 | 每步都是一次 API 调用 |
| 依赖 LLM 能力 | 小模型推理能力弱，ReAct 效果差 |
| 步数不可控 | 不知道要循环几次才能完成 |

### 应对策略

```python
# 1. 设置最大步数，防止无限循环
max_steps = 10

# 2. 检测重复行为
if current_action == last_action:
    print("检测到重复操作，强制结束")
    break

# 3. 设置超时
import time
start_time = time.time()
if time.time() - start_time > 60:  # 60 秒超时
    break
```

## 七、ReAct vs 其他模式

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| ReAct | 边想边做，一步一步来 | 大部分场景（最通用） |
| Plan-and-Execute | 先想好所有步骤，再执行 | 步骤明确的任务 |
| Reflexion | ReAct + 反思（做完回顾总结） | 需要高质量输出的任务 |
| Tree of Thoughts | 探索多条路径，选最优 | 复杂推理问题 |

实际项目中 90% 的场景用 ReAct 就够了。

---

## 总结

- ReAct = Thought（思考）+ Action（行动）+ Observation（观察）的循环
- 本质就是你在 Agent Skills Demo 2 写的 while 循环，加上显式的思考步骤
- 优点：可解释、能纠错、灵活
- 缺点：可能绕圈、成本高
- 是目前最主流的 Agent 模式，掌握它就掌握了 Agent 的核心
