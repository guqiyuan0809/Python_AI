## 一、为什么需要框架

你用纯代码实现 Agent 时，需要自己写：
- while 循环
- 工具注册和调用
- 消息管理
- 记忆管理
- 错误处理
- 日志记录

每个 Agent 项目都要重复写这些代码。框架把这些通用逻辑封装好了，你只需要关注业务逻辑。

```
纯代码：100+ 行实现 Agent 循环 + 50 行工具管理 + 50 行记忆管理 = 200+ 行基础代码
框架：  10-20 行配置 = 同样的功能
```

## 二、LangChain 简介

### LangChain 是什么

LangChain 是最流行的 LLM 应用开发框架，提供了构建 AI 应用的全套工具。

### 核心组件

```
LangChain 组件：
├── LLM / ChatModel    # 对接各种大模型（OpenAI、Claude、DeepSeek...）
├── Prompt Templates   # 提示词模板管理
├── Tools              # 工具定义和调用
├── Memory             # 记忆管理
├── Chains             # 链式调用（多步骤串联）
├── Agents             # Agent 实现
└── Output Parsers     # 输出格式化
```

### 用 LangChain 创建 Agent

```python
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor
from langchain.tools import Tool
from langchain import hub

# 1. 初始化 LLM
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key="your-key",
    base_url="https://api.deepseek.com"
)

# 2. 定义工具
def get_weather(city: str) -> str:
    return f"{city}：晴天，25°C"

tools = [
    Tool(
        name="get_weather",
        func=get_weather,
        description="查询指定城市的天气"
    )
]

# 3. 创建 Agent
prompt = hub.pull("hwchase17/react")  # 使用标准 ReAct prompt
agent = create_react_agent(llm, tools, prompt)

# 4. 创建执行器（处理循环、错误等）
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,      # 打印每步过程
    max_iterations=10  # 最大循环次数
)

# 5. 运行
result = agent_executor.invoke({"input": "北京天气怎么样？"})
print(result["output"])
```

对比你的纯代码实现，LangChain 帮你省掉了：
- while 循环（AgentExecutor 内部处理）
- 工具调用逻辑（自动解析和执行）
- 错误处理（内置重试机制）
- 日志输出（verbose=True）

## 三、LangGraph 简介

### LangGraph 是什么

LangGraph 是 LangChain 团队推出的 Agent 框架，用图（Graph）来定义 Agent 的工作流。

LangChain Agent 的问题：流程是固定的（ReAct 循环），不够灵活。
LangGraph 的解决：用节点（Node）和边（Edge）定义任意复杂的流程。

### 核心概念

```
Graph（图）= Nodes（节点）+ Edges（边）

Node：一个处理步骤（调用 LLM、执行工具、处理数据...）
Edge：节点之间的连接（定义执行顺序和条件分支）
State：在节点之间传递的状态数据
```

### 用 LangGraph 创建 Agent

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated

# 1. 定义状态
class AgentState(TypedDict):
    messages: list        # 对话消息
    next_action: str      # 下一步动作

# 2. 定义节点（每个节点是一个函数）
def call_llm(state: AgentState) -> AgentState:
    """调用 LLM 进行思考"""
    response = llm.invoke(state["messages"])
    state["messages"].append(response)
    
    if response.tool_calls:
        state["next_action"] = "use_tool"
    else:
        state["next_action"] = "end"
    return state

def use_tool(state: AgentState) -> AgentState:
    """执行工具调用"""
    # 执行工具，把结果加入 messages
    tool_result = execute_tools(state["messages"][-1].tool_calls)
    state["messages"].append(tool_result)
    state["next_action"] = "think"  # 执行完工具，回去思考
    return state

# 3. 构建图
graph = StateGraph(AgentState)

# 添加节点
graph.add_node("think", call_llm)
graph.add_node("use_tool", use_tool)

# 添加边（定义流程）
graph.set_entry_point("think")  # 入口：先思考

# 条件边：根据 next_action 决定下一步
graph.add_conditional_edges(
    "think",
    lambda state: state["next_action"],
    {
        "use_tool": "use_tool",  # 需要工具 → 去执行工具
        "end": END               # 不需要 → 结束
    }
)

graph.add_edge("use_tool", "think")  # 工具执行完 → 回去思考

# 4. 编译并运行
app = graph.compile()
result = app.invoke({"messages": [user_message], "next_action": "think"})
```

### LangGraph 的优势

```
LangChain Agent：
  think → tool → think → tool → ... → end
  （固定的 ReAct 循环，不好加自定义逻辑）

LangGraph：
  可以定义任意复杂的流程：
  
  think → 需要工具？ → yes → use_tool → think
                     → no  → 需要人工确认？ → yes → wait_human → think
                                            → no  → end
```

你可以加入：
- 人工确认节点（危险操作前让用户确认）
- 条件分支（根据任务类型走不同路径）
- 并行执行（多个工具同时调用）
- 子图（复杂流程拆成子流程）

## 四、LangChain vs LangGraph

| | LangChain Agent | LangGraph |
|--|--|--|
| 定位 | 快速创建标准 Agent | 创建复杂自定义 Agent |
| 灵活性 | 中等（固定模式） | 高（任意流程） |
| 学习成本 | 低 | 中 |
| 适用场景 | 简单 Agent、快速原型 | 复杂工作流、生产环境 |
| 代码量 | 少 | 中等 |

### 选择建议

```
简单 Agent（3-5 个工具，标准 ReAct）→ LangChain Agent 或纯代码
复杂 Agent（条件分支、人工确认、多步骤）→ LangGraph
学习阶段 → 先纯代码理解原理，再用框架
```

## 五、纯代码 vs 框架

### 什么时候用纯代码

- 学习阶段：理解 Agent 底层原理
- 简单 Agent：3-5 个工具，标准循环
- 特殊需求：框架不支持的自定义逻辑
- 轻量项目：不想引入额外依赖

### 什么时候用框架

- 生产项目：需要稳定性、错误处理、日志
- 复杂流程：条件分支、并行、人工确认
- 团队协作：框架提供统一的代码结构
- 快速开发：框架省去大量重复代码

### 建议路径

```
1. 先用纯代码实现 Agent（本章的 Demo）
   → 理解 while 循环、工具调用、记忆管理的底层原理

2. 再用 LangChain 实现同样的 Agent
   → 对比纯代码和框架的区别，理解框架帮你做了什么

3. 做复杂项目时用 LangGraph
   → 需要条件分支、人工确认等高级功能时再用
```

## 总结

- LangChain：最流行的 LLM 框架，快速创建标准 Agent
- LangGraph：用图定义复杂工作流，更灵活
- 学习路径：纯代码理解原理 → LangChain 快速开发 → LangGraph 复杂项目
- 本章 Demo 用纯代码实现，确保你理解底层原理
- 框架留到做实战项目时再深入学习
