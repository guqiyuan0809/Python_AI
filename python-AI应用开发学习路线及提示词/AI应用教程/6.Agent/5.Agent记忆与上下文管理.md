# Agent 记忆与上下文管理

> 你在 Agent Skills 阶段学了三种记忆（短期、工作、长期），这里讲 Agent 场景下更深入的记忆策略。

---

## 一、Agent 记忆 vs 普通对话记忆

普通对话：只需要记住用户说了什么。

Agent 执行任务：需要记住更多东西：

```
普通对话的 messages：
  user → assistant → user → assistant → ...

Agent 的 messages：
  user → assistant(tool_call) → tool(结果) → assistant(tool_call) → tool(结果) → ... → assistant(最终回答)
```

Agent 的 messages 增长速度远快于普通对话，因为每次工具调用都会增加至少 2 条消息（assistant 的 tool_call + tool 的结果）。

---

## 二、Agent 的上下文窗口问题

### 问题

LLM 有上下文窗口限制（比如 GPT-4 是 128K Token，DeepSeek 是 64K Token）。

Agent 执行复杂任务时，messages 可能很快就超出限制：

```
初始 prompt:          ~500 Token
每轮工具调用:         ~200-500 Token（tool_call + 结果）
10 轮循环后:          ~5000 Token
工具返回大量数据时:    可能一次就 10000+ Token
```

如果工具返回了一个大文件的内容或大量数据库查询结果，Token 会爆炸式增长。

### 解决方案

#### 1. 工具结果截断

```python
def execute_tool(tool_call):
    result = run_tool(tool_call)
    
    # 如果结果太长，截断
    MAX_RESULT_LENGTH = 2000
    if len(result) > MAX_RESULT_LENGTH:
        result = result[:MAX_RESULT_LENGTH] + f"\n...(结果过长，已截断，共 {len(result)} 字符)"
    
    return result
```

#### 2. 工具结果摘要

让 AI 先总结工具返回的大量数据，再继续推理：

```python
def summarize_tool_result(result, max_tokens=500):
    """让 AI 总结工具结果"""
    if len(result) < 1000:
        return result  # 短结果不需要总结
    
    summary_response = client.chat.completions.create(
        model=MODEL,
        messages=[
            {"role": "system", "content": "请用简洁的语言总结以下数据的关键信息："},
            {"role": "user", "content": result}
        ],
        max_tokens=max_tokens
    )
    return summary_response.choices[0].message.content
```

#### 3. 历史消息压缩

当 messages 太长时，把早期的对话压缩成摘要：

```python
def compress_history(messages, max_messages=20):
    """压缩历史消息"""
    if len(messages) <= max_messages:
        return messages
    
    system_msg = messages[0]
    old_messages = messages[1:-max_messages]  # 要压缩的旧消息
    recent_messages = messages[-max_messages:]  # 保留的近期消息
    
    # 让 AI 总结旧消息
    summary = summarize_messages(old_messages)
    
    # 用摘要替代旧消息
    return [
        system_msg,
        {"role": "system", "content": f"之前的对话摘要：{summary}"},
    ] + recent_messages
```

## 三、Agent 的任务记忆

Agent 执行多步任务时，需要一个专门的"任务状态"来跟踪进度：

```python
class TaskMemory:
    """Agent 任务记忆"""
    
    def __init__(self):
        self.task_description = ""     # 用户的原始任务
        self.plan = []                 # 执行计划
        self.completed_steps = []      # 已完成的步骤
        self.intermediate_results = {} # 中间结果
        self.current_step = 0          # 当前步骤
    
    def add_result(self, step_name: str, result: str):
        """记录步骤结果"""
        self.completed_steps.append(step_name)
        self.intermediate_results[step_name] = result
        self.current_step += 1
    
    def get_context(self) -> str:
        """生成任务上下文，加入 prompt"""
        context = f"当前任务：{self.task_description}\n"
        context += f"已完成步骤：\n"
        for step in self.completed_steps:
            result = self.intermediate_results[step]
            context += f"  - {step}: {result[:200]}\n"  # 截断长结果
        context += f"下一步：第 {self.current_step + 1} 步"
        return context
```

使用方式：

```python
task_memory = TaskMemory()
task_memory.task_description = "分析销售数据"

# 每完成一步，记录结果
task_memory.add_result("读取文件", "成功读取 sales.csv，共 1000 行")
task_memory.add_result("数据清洗", "去除 50 行空值，剩余 950 行")

# 把任务上下文加入 messages
context = task_memory.get_context()
messages.append({"role": "system", "content": context})
```

## 四、跨会话记忆

用户今天和 Agent 对话，明天再来，Agent 还能记住之前的内容。

### 实现方式

```python
import json
from datetime import datetime

# 本质是用本地 JSON 文件模拟数据库，简单场景够用，实际生产环境会换成 MySQL 或 Redis
class SessionMemory:
    """跨会话记忆管理"""
    
    def __init__(self, storage_path="sessions/"):
        self.storage_path = storage_path  # 存储目录，每个用户一个 JSON 文件
    
    def save_session(self, user_id: str, messages: list, metadata: dict = None):
        """保存会话：把本次对话追加写入用户的 JSON 文件"""
        session = {
            "user_id": user_id,
            "timestamp": datetime.now().isoformat(),  # 记录时间，方便后续按时间过滤
            "messages": messages,                      # 完整的对话记录
            "metadata": metadata or {}                 # 附加信息（如任务类型），没有就空字典
        }
        
        file_path = f"{self.storage_path}{user_id}_sessions.json"  # 每个用户独立一个文件
        try:
            with open(file_path, 'r', encoding='utf-8') as f:
                sessions = json.load(f)   # 读取已有的历史会话
        except (FileNotFoundError, json.JSONDecodeError):
            sessions = []                 # 文件不存在或格式异常，从空列表开始
        
        sessions.append(session)          # 把本次会话追加进去
        
        with open(file_path, 'w', encoding='utf-8') as f:
            json.dump(sessions, f, ensure_ascii=False, indent=2)  # 写回文件，中文不转义
    
    def load_recent_sessions(self, user_id: str, count: int = 3) -> list:
        """加载最近 N 次会话（默认取最近 3 次，避免上下文过长）"""
        file_path = f"{self.storage_path}{user_id}_sessions.json"
        try:
            with open(file_path, 'r', encoding='utf-8') as f:
                sessions = json.load(f)
            return sessions[-count:]      # 切片取最后 N 条
        except (FileNotFoundError, json.JSONDecodeError):
            return []                     # 文件不存在说明是新用户，返回空列表
    
    def build_context_from_history(self, user_id: str) -> str:
        """把历史会话整理成文字摘要，塞进 System Prompt 让 AI 知道用户背景"""
        sessions = self.load_recent_sessions(user_id)
        if not sessions:
            return ""  # 新用户没有历史，返回空字符串
        
        context = "用户的历史会话摘要：\n"
        for session in sessions:
            time = session["timestamp"]
            # 只取用户消息（过滤掉 AI 的回复），最多取前 3 条避免太长
            user_msgs = [m["content"] for m in session["messages"] if m["role"] == "user"]
            context += f"- {time}: 讨论了 {', '.join(user_msgs[:3])}\n"
        
        return context  # 最终作为 System Prompt 的一部分传给 AI
```

### 实际项目中的存储选择

| 存储方式 | 适用场景 | 说明 |
|---------|---------|------|
| JSON 文件 | 学习、原型开发 | 简单但不适合生产 |
| SQLite | 单机小项目 | 轻量级数据库，不需要安装 |
| MySQL/PostgreSQL | 生产环境 | 结构化存储，支持复杂查询 |
| Redis | 需要高速读写 | 适合缓存热数据 |
| 向量数据库 | 需要语义检索 | 按相似度找相关历史（和 RAG 一样） |

## 五、记忆策略的组合使用

实际 Agent 项目中，通常组合多种记忆策略：

```
┌─────────────────────────────────────────┐
│              Agent 记忆体系              │
│                                         │
│  ┌─────────────┐  ┌─────────────────┐  │
│  │  短期记忆    │  │  任务记忆        │  │
│  │  messages   │  │  当前任务状态     │  │
│  │  当前对话    │  │  中间结果        │  │
│  └─────────────┘  └─────────────────┘  │
│                                         │
│  ┌─────────────┐  ┌─────────────────┐  │
│  │  工作记忆    │  │  长期记忆        │  │
│  │  滑动窗口    │  │  用户偏好(MySQL) │  │
│  │  控制成本    │  │  历史会话(MySQL) │  │
│  └─────────────┘  │  语义检索(向量DB) │  │
│                    └─────────────────┘  │
└─────────────────────────────────────────┘
```

一个典型的 Agent 启动流程：

```python
def start_agent_session(user_id, user_message):
    # 1. 加载长期记忆（用户偏好）
    user_profile = load_user_profile(user_id)
    
    # 2. 加载历史会话摘要
    history_context = session_memory.build_context_from_history(user_id)
    
    # 3. 构建 system prompt（包含记忆）
    system_prompt = f"""你是一个助手。
    
用户信息：{user_profile}
历史会话：{history_context}
"""
    
    # 4. 初始化短期记忆
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_message}
    ]
    
    # 5. 初始化任务记忆
    task_memory = TaskMemory()
    task_memory.task_description = user_message
    
    # 6. 开始 Agent 循环
    result = agent_loop(messages, task_memory)
    
    # 7. 保存会话到长期记忆
    session_memory.save_session(user_id, messages)
    
    return result
```

## 总结

- Agent 的 messages 增长速度远快于普通对话，需要主动管理
- 三种控制策略：工具结果截断、结果摘要、历史压缩
- 任务记忆：跟踪多步任务的进度和中间结果
- 跨会话记忆：让 Agent 记住用户的历史交互
- 实际项目中组合使用多种记忆策略
- 存储选择：学习用 JSON 文件，生产用 MySQL + 向量数据库
