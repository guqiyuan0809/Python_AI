# Agent 评估与调试

> Agent 比普通 AI 应用更难调试，因为它的行为是动态的、不确定的。
> 掌握评估和调试方法，才能做出稳定可靠的 Agent。

---

## 一、Agent 为什么难调试

普通 AI 应用：输入固定 → 输出基本固定，出了问题看 prompt 就行。

Agent：同样的输入，可能走不同的路径、调用不同的工具、循环不同的次数。

```
同一个问题 "分析销售数据"：
  第 1 次运行：读文件 → 清洗 → 分析 → 出图 → 报告（5 步）
  第 2 次运行：读文件 → 清洗 → 发现异常 → 重新清洗 → 分析 → 报告（6 步）
  第 3 次运行：读文件 → 格式错误 → 换方法读 → 清洗 → 分析 → 报告（6 步）
```

每次路径可能不同，这就是 Agent 调试的难点。

## 二、Agent 的日志系统

调试 Agent 最重要的工具是日志。记录每一步的详细信息：
# 记录的内容一般是:步骤，工具调用，token使用情况，错误，输出执行摘要
# 日志形式一般是控制台，写入日志文件

```python
import logging
from datetime import datetime

class AgentLogger:
    """Agent 专用日志"""
    
    def __init__(self, log_file="agent.log"):
        self.logger = logging.getLogger("agent")
        self.logger.setLevel(logging.DEBUG)
        
        # 文件日志
        handler = logging.FileHandler(log_file, encoding='utf-8')
        handler.setFormatter(logging.Formatter(
            '%(asctime)s [%(levelname)s] %(message)s'
        ))
        self.logger.addHandler(handler)
        
        # 控制台日志
        console = logging.StreamHandler()
        console.setFormatter(logging.Formatter(
            '[%(levelname)s] %(message)s'
        ))
        self.logger.addHandler(console)
        
        self.step_count = 0
        self.total_tokens = 0
    
    def log_step(self, step_type: str, content: str):
        """记录每一步"""
        self.step_count += 1
        self.logger.info(f"Step {self.step_count} [{step_type}]: {content}")
    
    def log_tool_call(self, tool_name: str, args: dict, result: str):
        """记录工具调用"""
        self.logger.info(f"Tool Call: {tool_name}({args})")
        self.logger.info(f"Tool Result: {result[:500]}")  # 截断长结果
    
    def log_token_usage(self, usage):
        """记录 Token 使用"""
        if usage:
            tokens = usage.total_tokens
            self.total_tokens += tokens
            self.logger.info(f"Tokens: {tokens} (累计: {self.total_tokens})")
    
    def log_error(self, error: str):
        """记录错误"""
        self.logger.error(f"Error: {error}")
    
    def summary(self):
        """输出执行摘要"""
        self.logger.info(f"=== 执行摘要 ===")
        self.logger.info(f"总步数: {self.step_count}")
        self.logger.info(f"总 Token: {self.total_tokens}")
```

在 Agent 循环中使用：

```python
logger = AgentLogger()

def agent_loop(user_message, tools):
    logger.log_step("START", f"用户输入: {user_message}")
    
    for step in range(max_steps):
        response = client.chat.completions.create(...)
        logger.log_token_usage(response.usage)
        
        message = response.choices[0].message
        
        if message.tool_calls:
            for tool_call in message.tool_calls:
                result = execute_tool(tool_call)
                logger.log_tool_call(
                    tool_call.function.name,
                    tool_call.function.arguments,
                    result
                )
        else:
            logger.log_step("FINISH", f"最终回答: {message.content[:200]}")
            break
    
    logger.summary()
```

## 三、常见问题与排查

### 问题 1：Agent 无限循环

症状：Agent 反复调用同一个工具，不停下来。

原因：
- AI 没有意识到任务已经完成
- 工具返回的结果不够明确
- System Prompt 没有告诉 AI 什么时候该停

排查方法：
```python
# 1. 检查日志，看 AI 每步在想什么
# 2. 检查工具返回结果是否清晰
# 3. 检查是否有重复的 tool_call

# 解决方案
# 方案 1：设置最大步数
max_steps = 10

# 方案 2：检测重复调用
last_calls = []
if current_call in last_calls[-3:]:  # 最近 3 次有重复
    break

# 方案 3：在 prompt 中明确告诉 AI
system_prompt += "\n当你获得了足够的信息来回答用户问题时，直接给出最终回答，不要继续调用工具。"
```

### 问题 2：Agent 选错工具

症状：用户问天气，AI 却调用了计算器。

原因：
- 工具的 description 写得不够清楚
- 工具太多，AI 搞混了
- 用户的问题表述模糊

排查方法：
```python
# 1. 检查日志中 AI 的 tool_call 选择
# 2. 检查工具 description 是否有歧义
# 3. 测试：只保留相关工具，看 AI 是否能正确选择

# 解决方案
# 方案 1：优化 description
# 方案 2：减少工具数量
# 方案 3：在 system prompt 中给出工具使用指南
```

### 问题 3：Agent 回答质量差

症状：Agent 完成了任务，但回答不够好。

原因：
- 工具返回的数据不够
- AI 没有充分利用工具结果
- System Prompt 对输出格式没有要求

排查方法：
```python
# 1. 检查每步工具返回的数据是否完整
# 2. 检查 AI 是否跳过了某些必要步骤
# 3. 优化 system prompt，明确输出要求

system_prompt += """
回答要求：
1. 基于工具返回的实际数据
2. 包含具体数字和事实
3. 结构清晰，分点列出
4. 如果数据不足，明确告诉用户
"""
```

### 问题 4：Token 成本过高

症状：一次任务消耗了大量 Token。

排查方法：
```python
# 1. 检查日志中的 Token 使用情况
# 2. 找出哪一步消耗最多（通常是工具返回大量数据）
# 3. 优化

# 解决方案
# 方案 1：截断工具返回结果
# 方案 2：压缩历史消息
# 方案 3：用更便宜的模型处理简单步骤
```

## 四、Agent 评估指标

怎么判断你的 Agent 好不好？

### 1. 任务完成率

```
完成率 = 成功完成的任务数 / 总任务数

目标：> 80%（简单任务），> 60%（复杂任务）
```

### 2. 平均步数

```
平均步数 = 总步数 / 任务数

步数越少越好（说明 Agent 效率高）
如果平均步数 > 8，可能需要优化
```

### 3. Token 成本

```
平均成本 = 总 Token 数 / 任务数

关注每个任务的成本是否在可接受范围内
```

### 4. 错误率

```
错误率 = 出错的步骤数 / 总步骤数

包括：选错工具、参数错误、工具执行失败
```

### 简单的评估脚本

```python
def evaluate_agent(test_cases):
    """评估 Agent 表现"""
    results = {
        "total": len(test_cases),     # 总任务数
        "success": 0,                  # → 对应指标：任务完成率
        "failed": 0,
        "total_steps": 0,             # → 对应指标：平均步数（预留字段，需配合 AgentLogger 累加）
        "total_tokens": 0,            # → 对应指标：Token 成本（预留字段，需配合 AgentLogger 累加）
        "errors": []                   # → 对应指标：错误率 / 失败详情
    }
    
    for case in test_cases:
        try:
            result = agent_loop(case["input"], tools)  # 运行 Agent，拿到最终回答
            
            # 简单判断：回答里包含预期关键词就算成功
            if case["expected_keyword"] in result:
                results["success"] += 1
            else:
                results["failed"] += 1
                results["errors"].append({
                    "input": case["input"],
                    "expected": case["expected_keyword"],
                    "got": result[:200]               # 截取前 200 字，避免日志太长
                })
        except Exception as e:
            results["failed"] += 1                     # 报错也算失败
            results["errors"].append({"input": case["input"], "error": str(e)})
    
    # 输出评估报告
    print(f"完成率: {results['success']}/{results['total']}")  # → 任务完成率
    print(f"失败案例: {results['errors']}")                     # → 具体哪些失败了
    return results

# 测试用例：input 是用户输入，expected_keyword 是回答中应包含的关键词
test_cases = [
    {"input": "北京天气怎么样", "expected_keyword": "北京"},     # 测试工具调用
    {"input": "123 * 456", "expected_keyword": "56088"},         # 测试计算工具
    {"input": "什么是 Python", "expected_keyword": "Python"},    # 测试纯对话（不需要工具）
]

evaluate_agent(test_cases)
```

## 五、调试技巧总结

| 技巧 | 说明 |
|------|------|
| 打印每步的 Thought | 看 AI 在想什么 |
| 记录所有 tool_call | 看 AI 选了什么工具、传了什么参数 |
| 记录 Token 使用 | 找出成本瓶颈 |
| 设置最大步数 | 防止无限循环 |
| 检测重复调用 | 发现 Agent 卡住的情况 |
| 用简单任务先测试 | 确保基本功能正常再测复杂任务 |
| 逐步增加工具 | 先 1-2 个工具跑通，再加更多 |

---

## 总结

- Agent 调试的核心是日志：记录每步的思考、工具调用、结果
- 常见问题：无限循环、选错工具、回答质量差、成本高
- 每个问题都有对应的排查方法和解决方案
- 评估指标：完成率、平均步数、Token 成本、错误率
- 先用简单任务调通，再逐步增加复杂度
