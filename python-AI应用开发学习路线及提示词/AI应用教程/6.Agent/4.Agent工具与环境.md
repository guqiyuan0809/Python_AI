# Agent 工具与环境

> 工具是 Agent 的"手脚"，没有工具的 Agent 只是一个聊天机器人。
> 工具设计的好坏直接影响 Agent 的表现。

---

## 一、工具设计原则

你在 Agent Skills 阶段学了 Function Calling 的基本用法，这里讲怎么设计好的工具。

### 原则 1：单一职责

每个工具只做一件事：

```python
# ❌ 不好：一个工具做太多事
def data_tool(action, file_path=None, sql=None, chart_type=None):
    if action == "read": ...
    elif action == "query": ...
    elif action == "chart": ...

# ✅ 好：每个工具职责清晰
def read_file(file_path: str) -> str: ...
def query_database(sql: str) -> str: ...
def create_chart(data: str, chart_type: str) -> str: ...
```

为什么？AI 更容易理解和选择职责单一的工具。一个工具做太多事，AI 容易搞混参数。

### 原则 2：描述写清楚

description 是 AI 选择工具的唯一依据，写得越清楚，AI 选得越准：

```python
# ❌ 不好：描述模糊
{"name": "search", "description": "搜索"}

# ✅ 好：描述具体
{"name": "search_web", "description": "搜索互联网获取最新信息，适用于需要实时数据的问题，如新闻、价格、天气等"}
```

### 原则 3：参数要有说明

```python
# ❌ 不好：参数没说明
"parameters": {
    "properties": {
        "q": {"type": "string"}
    }
}

# ✅ 好：参数有详细说明
"parameters": {
    "properties": {
        "query": {
            "type": "string",
            "description": "搜索关键词，尽量具体，如'北京今天天气'而不是'天气'"
        }
    }
}
```

### 原则 4：返回结果要有用

工具返回的结果要包含足够信息，让 AI 能基于结果做判断：

```python
# ❌ 不好：返回太简单
def get_weather(city):
    return "25"  # AI 不知道这是温度还是什么

# ✅ 好：返回信息完整
def get_weather(city):
    return f"{city}：晴天，温度 25°C，湿度 40%，空气质量良好"
```

## 三、工具的错误处理

Agent 调用工具时可能出错，好的错误处理能让 Agent 自我纠正：

### 返回有意义的错误信息

```python
def query_database(sql: str) -> str:
    """执行 SQL 查询"""
    try:
        result = db.execute(sql)
        if not result:
            return "查询成功，但没有找到匹配的数据"
        return json.dumps(result, ensure_ascii=False)
    except Exception as e:
        # 返回有意义的错误信息，AI 可以据此调整
        return f"SQL 执行错误: {str(e)}。请检查表名和字段名是否正确。"   #错误信息作为tool 结果返回给了 AI
```

为什么不直接抛异常？因为 Agent 的循环需要继续运行，把错误信息作为 tool 结果返回给 AI，AI 可以根据错误信息调整策略（比如换个 SQL 重试）。

### 工具超时处理

```python
import signal

def run_with_timeout(func, args, timeout=30):
    """给工具执行加超时限制"""
    try:
        # 简单方式：用 threading
        import threading
        result = [None]
        error = [None]
        
        def target():
            try:
                result[0] = func(*args)
            except Exception as e:
                error[0] = str(e)
        
        thread = threading.Thread(target=target)
        thread.start()
        thread.join(timeout)
        
        if thread.is_alive():
            return f"工具执行超时（{timeout}秒），请简化操作或拆分任务"
        if error[0]:
            return f"工具执行错误: {error[0]}"
        return result[0]
    except Exception as e:
        return f"执行异常: {str(e)}"
```

## 四、工具安全性

Agent 能执行代码、操作数据库，安全性很重要：

### 1. 代码执行沙箱

如果 Agent 需要执行用户相关的代码，必须在沙箱中运行：

```python
# ❌ 危险：直接 eval/exec
def run_code(code: str) -> str:
    return str(eval(code))  # 用户可能注入恶意代码

# ✅ 安全：限制可用的函数和模块
def run_code_safe(code: str) -> str:
    allowed_modules = {'math', 'statistics', 'json'}
    # 检查代码中是否有危险操作
    dangerous_keywords = ['import os', 'import sys', 'exec(', 'eval(', '__import__']
    for keyword in dangerous_keywords:
        if keyword in code:
            return f"安全限制：不允许使用 {keyword}"
    
    # 在受限环境中执行
    safe_globals = {"__builtins__": {}}
    try:
        exec(code, safe_globals)
        return str(safe_globals.get('result', '执行完成'))
    except Exception as e:
        return f"执行错误: {e}"
```

### 2. 数据库权限控制

```python
# ❌ 危险：允许任意 SQL
def query_database(sql: str) -> str:
    return db.execute(sql)  # 可能执行 DROP TABLE

# ✅ 安全：只允许 SELECT
def query_database(sql: str) -> str:
    sql_upper = sql.strip().upper()
    if not sql_upper.startswith("SELECT"):
        return "安全限制：只允许 SELECT 查询，不允许修改数据"
    return db.execute(sql)
```

## 五、工具数量的权衡

### 工具太少

Agent 能力有限，很多任务做不了。

### 工具太多

AI 容易选错工具，推理负担重，Token 成本高（每个工具描述都占 Token）。

### 建议

| 场景 | 推荐工具数量 |
|------|------------|
| 简单 Agent（聊天 + 几个功能） | 3-5 个 |
| 中等 Agent（数据分析、文档处理） | 5-10 个 |
| 复杂 Agent（全能助手） | 10-15 个 |
| 超过 15 个 | 考虑拆成多个 Agent |

如果工具超过 15 个，AI 选择准确率会明显下降。这时候应该考虑：
- 把工具分组，用多个专业 Agent
- 或者用一个"路由 Agent"先判断任务类型，再分发给专业 Agent

## 六、工具注册的最佳实践

### 用装饰器简化工具注册

```python
# 工具注册表
tool_registry = {}

def tool(name: str, description: str):
    """工具注册装饰器"""
    def decorator(func):
        tool_registry[name] = {
            "function": func,
            "description": description,
            "parameters": extract_parameters(func)  # 从函数签名自动提取参数
        }
        return func
    return decorator

@tool("get_weather", "查询指定城市的实时天气")
def get_weather(city: str) -> str:
    ...

@tool("calculator", "计算数学表达式")
def calculator(expression: str) -> str:
    ...

# 自动生成 tools 参数和 available_functions
tools = [format_tool(t) for t in tool_registry.values()]
available_functions = {name: info["function"] for name, info in tool_registry.items()}
```

这样添加新工具只需要加一个 `@tool` 装饰器，不用手动维护 tools 列表和 available_functions 字典。

## 总结

- 工具设计四原则：单一职责、描述清楚、参数有说明、返回有用信息
- 错误处理：返回有意义的错误信息，让 AI 能自我纠正
- 安全性：代码沙箱、SQL 权限控制、API 频率限制
- 工具数量：5-10 个最佳，超过 15 个考虑拆分
- 用装饰器简化工具注册，减少重复代码
