# LangGraph 多 Agent

## 为什么用 LangGraph 做多 Agent

LangGraph 是 LangChain 团队开发的图工作流框架。它用"图"来描述 Agent 之间的协作关系：
- **节点（Node）**：每个 Agent 或处理步骤
- **边（Edge）**：Agent 之间的连接和流转逻辑
- **状态（State）**：所有 Agent 共享的数据

相比 CrewAI，LangGraph 更灵活，适合需要复杂流转逻辑的场景。

## 核心概念

### State（共享状态）

所有节点共享一个状态对象，每个节点可以读取和修改状态：

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph
import operator

class TeamState(TypedDict):
    messages: Annotated[list, operator.add]
    task: str
    research_result: str
    draft: str
    final_output: str
```

### Node（节点 = Agent）

每个节点是一个函数，接收状态、返回更新：

```python
def researcher_node(state: TeamState):
    """研究员节点：根据任务进行研究"""
    task = state["task"]
    # 调用 LLM 进行研究
    result = llm.invoke(f"请研究以下主题：{task}")
    return {"research_result": result.content}

def writer_node(state: TeamState):
    """写作者节点：根据研究结果撰写内容"""
    research = state["research_result"]
    draft = llm.invoke(f"根据以下研究结果撰写文章：{research}")
    return {"draft": draft.content}
```

### Edge（边 = 流转逻辑）

定义节点之间的连接，支持条件路由：

```python
from langgraph.graph import END

# 构建图
graph = StateGraph(TeamState)

# 添加节点
graph.add_node("researcher", researcher_node)
graph.add_node("writer", writer_node)
graph.add_node("reviewer", reviewer_node)

# 添加边（流转逻辑）
graph.set_entry_point("researcher")
graph.add_edge("researcher", "writer")
graph.add_edge("writer", "reviewer")

# 条件边：审核通过则结束，不通过则回到写作者
graph.add_conditional_edges(
    "reviewer",
    review_decision,  # 返回 "pass" 或 "revise"
    {"pass": END, "revise": "writer"}
)

app = graph.compile()
```

## LangGraph 多 Agent 模式

### Supervisor 模式（层级）

一个 Supervisor 节点决定下一步由谁执行：

```python
def supervisor_node(state):
    """主管决定下一步由谁执行"""
    response = llm.invoke(
        "根据当前进度，下一步应该由谁执行？"
        "选项：researcher / writer / FINISH"
    )
    return {"next": response.content}
```

### 协作模式（对等）

Agent 之间直接传递信息，没有中心管理者。

## LangGraph vs CrewAI

| 维度 | LangGraph | CrewAI |
|------|-----------|--------|
| 灵活性 | 高（自定义图结构） | 中（预设模式） |
| 上手难度 | 较高 | 低 |
| 流转控制 | 精细（条件边、循环） | 简单（串行/层级） |
| 状态管理 | 显式共享状态 | 隐式消息传递 |
| 适用场景 | 复杂工作流 | 快速搭建团队 |
