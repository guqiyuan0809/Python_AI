# 多 Agent 协作概述

## 什么是多 Agent 协作

多 Agent 协作是指多个 AI Agent 各自扮演不同角色，通过分工合作来完成复杂任务。每个 Agent 有自己的专长、工具和职责，就像一个团队中不同岗位的成员。

## 为什么需要多 Agent

单个 Agent 的局限性：
- 单个 Agent 处理复杂任务时，Prompt 会变得很长，容易"迷失"
- 一个 Agent 很难同时精通多个领域
- 复杂任务需要不同的思维方式（分析、执行、审查）

多 Agent 的优势：
- **专业化**：每个 Agent 专注一个领域，Prompt 更精准
- **可扩展**：新增能力只需加一个 Agent，不影响现有逻辑
- **质量控制**：可以设置"审查 Agent"来检查其他 Agent 的输出
- **并行处理**：独立子任务可以同时执行，提高效率

## 多 Agent vs 单 Agent 对比

| 维度 | 单 Agent | 多 Agent |
|------|---------|---------|
| 适用场景 | 单一领域、简单任务 | 跨领域、复杂任务 |
| Prompt 复杂度 | 一个大 Prompt | 多个小 Prompt |
| 调试难度 | 低 | 高 |
| Token 消耗 | 较少 | 较多 |
| 开发成本 | 低 | 高 |
| 灵活性 | 低 | 高 |

## 典型应用场景

1. **软件开发团队**：产品经理 Agent → 开发 Agent → 测试 Agent → 代码审查 Agent
2. **内容创作**：研究 Agent → 写作 Agent → 编辑 Agent
3. **数据分析**：数据清洗 Agent → 分析 Agent → 可视化 Agent → 报告 Agent
4. **客服系统**：分类 Agent → 专业客服 Agent → 质检 Agent
