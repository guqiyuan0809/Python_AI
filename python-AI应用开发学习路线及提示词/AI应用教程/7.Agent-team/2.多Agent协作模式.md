# 多 Agent 协作模式

## 三种核心协作模式

### 1. 串行模式（Sequential / Pipeline）

Agent 按顺序依次执行，前一个的输出是后一个的输入。

```
Agent A → Agent B → Agent C → 最终结果
```

**特点**：
- 简单直观，容易理解和调试
- 每个 Agent 只需关注自己的输入输出
- 缺点：速度慢，一个 Agent 卡住整个流程都停

**适用场景**：
- 内容创作流水线（研究 → 写作 → 审核）
- 数据处理管道（清洗 → 分析 → 可视化）
- 代码生成流程（设计 → 编码 → 测试）

### 2. 并行模式（Parallel）

多个 Agent 同时处理不同子任务，最后汇总结果。

```
         ┌→ Agent A ─┐
任务拆分 ─┼→ Agent B ─┼→ 结果汇总
         └→ Agent C ─┘
```

**特点**：
- 速度快，多个任务同时进行
- 需要一个"协调者"来拆分任务和汇总结果
- 缺点：子任务之间不能有依赖关系

**适用场景**：
- 多维度分析（同时分析市场、技术、财务）
- 多语言翻译（同时翻译成多种语言）
- 多源信息收集（同时搜索不同数据源）

### 3. 层级模式（Hierarchical / Manager）

一个"管理者 Agent"负责分配任务、监督执行、汇总结果。

```
              Manager Agent
            /       |       \
     Worker A   Worker B   Worker C
```

**特点**：
- 最接近真实团队的工作方式
- Manager 负责决策和协调，Worker 负责执行
- Manager 可以根据中间结果动态调整计划
- 缺点：Manager 的能力决定了整个系统的上限

**适用场景**：
- 复杂项目管理（Manager 分配任务给不同专家）
- 多步骤问题求解（Manager 决定下一步该做什么）
- 需要动态决策的场景

## 协作模式选择指南

| 场景特征 | 推荐模式 |
|---------|---------|
| 任务有明确的先后顺序 | 串行 |
| 子任务相互独立 | 并行 |
| 需要动态决策和协调 | 层级 |
| 简单流水线 + 部分并行 | 混合（串行 + 并行） |

## 通信机制

Agent 之间如何传递信息：

1. **直接传递**：前一个 Agent 的输出直接作为下一个的输入
2. **共享状态**：所有 Agent 读写同一个状态对象（LangGraph 的方式）
3. **消息队列**：Agent 通过消息发送/接收信息（CrewAI 的方式）
4. **黑板模式**：所有 Agent 读写一个公共"黑板"，按需获取信息
