# AutoGen 框架

## 什么是 AutoGen

AutoGen 是微软开源的多 Agent 对话框架。它的核心理念是：Agent 之间通过"对话"来协作，就像人类团队开会讨论一样。

## 核心特点

- **对话驱动**：Agent 之间通过消息来回交流，直到达成共识
- **人机协作**：可以在 Agent 对话中插入人类参与
- **代码执行**：内置安全的代码执行环境
- **灵活组合**：支持两人对话、群聊等多种模式

## 核心概念

### ConversableAgent（可对话 Agent）

AutoGen 中所有 Agent 的基类：

```python
from autogen import ConversableAgent

assistant = ConversableAgent(
    name="助手",
    system_message="你是一个有帮助的AI助手。",
    llm_config={"model": "gpt-4", "api_key": "your-key"}
)

user_proxy = ConversableAgent(
    name="用户代理",
    human_input_mode="NEVER",  # 不需要人类输入
    code_execution_config={"work_dir": "coding"}
)
```

### 两人对话模式

最简单的协作：两个 Agent 来回对话：

```python
# 用户代理发起对话，助手回复
user_proxy.initiate_chat(
    assistant,
    message="帮我写一个Python函数，计算斐波那契数列前N项。"
)
```

### 群聊模式（GroupChat）

多个 Agent 在一个"群"里讨论：

```python
from autogen import GroupChat, GroupChatManager

# 创建多个专业 Agent
coder = ConversableAgent(name="程序员", ...)
reviewer = ConversableAgent(name="审查员", ...)
tester = ConversableAgent(name="测试员", ...)

# 创建群聊
group_chat = GroupChat(
    agents=[coder, reviewer, tester],
    messages=[],
    max_round=10
)

# 群聊管理者（决定谁发言）
manager = GroupChatManager(groupchat=group_chat)
```

## AutoGen 的工作流程

```
1. 定义 Agent（角色、系统提示、LLM 配置）
2. 选择对话模式（两人对话 / 群聊）
3. 发起对话（initiate_chat）
4. Agent 自动来回交流，直到任务完成或达到最大轮次
5. 获取对话历史和最终结果
```


## AutoGen vs CrewAI vs LangGraph

| 维度 | AutoGen | CrewAI | LangGraph |
|------|---------|--------|-----------|
| 核心理念 | 对话协作 | 角色扮演团队 | 图工作流 |
| 协作方式 | 消息来回 | 任务分配 | 状态流转 |
| 人类参与 | 原生支持 | 有限 | 需自定义 |
| 代码执行 | 内置沙箱 | 需配置 | 需自定义 |
| 上手难度 | 中 | 低 | 高 |
| 适用场景 | 讨论式任务 | 流水线任务 | 复杂工作流 |

## 使用建议

- 快速搭建多 Agent demo → 选 CrewAI
- 需要精细控制流程 → 选 LangGraph
- 需要 Agent 之间深度讨论 → 选 AutoGen
- 生产环境 → LangGraph（更可控、更稳定）
