# MCP 实践指南

> 本文档讲怎么用现成的 MCP Server，以及怎么自己写一个 MCP Server。

---

## 一、使用现成的 MCP Server

### 在 Claude Desktop / Cursor 中配置

这些 AI 工具已经内置了 MCP Client，你只需要配置 MCP Server 就行。

**配置文件位置**：
- Claude Desktop（Windows）：`%APPDATA%\Claude\claude_desktop_config.json`
- Cursor：Settings → MCP

**配置示例（文件系统 MCP Server）**：
```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "D:/你的项目目录"
      ]
    }
  }
}
```

配置完成后，AI 就能读取你指定目录下的文件了。

## 二、自己写 MCP Server（Python）

### 安装依赖

```bash
pip install mcp
```

### 最小 MCP Server 示例

```python
# my_mcp_server.py
from mcp.server.fastmcp import FastMCP

# 创建 MCP Server
mcp = FastMCP("我的工具箱")


# 定义工具：AI 可以调用这个函数
@mcp.tool()
def add(a: int, b: int) -> int:
    """两个数相加"""
    return a + b


@mcp.tool()
def get_weather(city: str) -> str:
    """查询城市天气（模拟）"""
    # 实际项目中这里调真实天气 API
    weather_data = {
        "北京": "晴天，25°C",
        "上海": "多云，22°C",
        "深圳": "阵雨，28°C",
    }
    return weather_data.get(city, f"暂无 {city} 的天气数据")


# 启动 Server
if __name__ == "__main__":
    mcp.run()
```

**就这么简单** — 用 `@mcp.tool()` 装饰器把普通函数变成 AI 可调用的工具。

### 关键点

1. **函数名**：AI 会看到函数名，用来判断什么时候调用
2. **Docstring**：AI 会读 docstring 来理解这个工具的作用（必须写！）
3. **类型注解**：AI 根据参数类型知道传什么值（必须写！）
4. **返回值**：工具执行的结果会返回给 AI，AI 基于结果继续回答

## 三、运行和测试 MCP Server

### 方式一：用 MCP Inspector 测试（推荐）

MCP 官方提供了一个测试工具 MCP Inspector：

```bash
npx @modelcontextprotocol/inspector python my_mcp_server.py
```

会打开一个 Web 界面，可以看到 Server 暴露了哪些工具，手动调用测试。

### 方式二：在 Claude Desktop 中使用

在 `claude_desktop_config.json` 中配置你自己的 Server：

```json
{
  "mcpServers": {
    "my-tools": {
      "command": "python",
      "args": ["D:/桌面代码文件/AI应用/3.MCP/my_mcp_server.py"]
    }
  }
}
```

重启 Claude Desktop 后，它就能使用你定义的工具了。

## 四、更实用的 MCP Server 示例

### 数据库查询工具

```python
@mcp.tool()
def query_users(city: str) -> str:
    """查询指定城市的用户数量"""
    import sqlite3
    conn = sqlite3.connect("app.db")
    cursor = conn.execute(
        "SELECT COUNT(*) FROM users WHERE city = ?", (city,)
    )
    count = cursor.fetchone()[0]
    conn.close()
    return f"{city} 有 {count} 个用户"
```

### 文件搜索工具

```python
import os

@mcp.tool()
def search_files(directory: str, keyword: str) -> str:
    """在指定目录中搜索包含关键词的文件"""
    results = []
    for root, dirs, files in os.walk(directory):
        for file in files:
            if keyword.lower() in file.lower():
                results.append(os.path.join(root, file))
    if results:
        return "\n".join(results[:20])  # 最多返回 20 个
    return f"未找到包含 '{keyword}' 的文件"
```

### 暴露资源（Resource）

```python
@mcp.resource("config://app")
def get_app_config() -> str:
    """返回应用配置信息"""
    return "当前环境: development, 数据库: SQLite, 端口: 8000"
```

## 五、MCP Server 开发要点

| 要点 | 说明 |
|------|------|
| **Docstring 必须写** | AI 靠 docstring 理解工具作用，不写 AI 不知道什么时候用 |
| **类型注解必须写** | AI 靠类型注解知道传什么参数 |
| **返回字符串** | 工具的返回值最终要变成文本给 AI 看，建议直接返回 str |
| **做好错误处理** | 工具执行可能出错，try-except 包住，返回友好的错误信息 |
| **控制权限** | 不要暴露危险操作（删库、删文件），做好安全限制 |

## 六、MCP 生态现状

| 支持 MCP 的 AI 应用 | 说明 |
|---------------------|------|
| Claude Desktop | Anthropic 官方客户端，MCP 支持最完整 |
| Cursor | AI 编程 IDE，支持 MCP |
| Windsurf | AI 编程 IDE，支持 MCP |
| Continue | 开源 AI 编程插件，支持 MCP |

**注意**：目前 MCP 主要用在 AI IDE 和桌面应用中。在你自己写的 Python 后端应用里，更常用的是直接用 Function Calling / LangChain Tools，而不是 MCP 协议。MCP 的价值是**跨应用复用**。

---

## 七、MCP vs 直接写 Function Calling

| 场景 | 推荐方式 |
|------|---------|
| 给 Claude Desktop / Cursor 加功能 | MCP Server |
| 自己写 AI 后端应用（FastAPI） | Function Calling / LangChain Tools |
| 需要同一工具被多个 AI 应用共用 | MCP Server |
| 做 Agent 项目 | LangChain Tools（后面第 5-6 阶段会学） |

---

## 八、总结

- MCP Server 写起来很简单：`@mcp.tool()` 装饰器 + 函数 + docstring
- 现成的 MCP Server 很多，大部分场景不需要自己写
- MCP 的核心价值是**标准化** — 写一次，所有 MCP Host 都能用
- 后面做 Agent 项目时，更常用 LangChain Tools，但理解 MCP 有助于理解工具调用的本质
