# Agent 与 AgentExecutor — 组装与运行

> 上一章你学会了写 Tools。现在要把 Tools 和 LLM 装到一起，让它跑起来。
> 这一章是 Agent 工程的"主装配线"。

---

## 一、三个核心对象

```
┌──────────────────────────────────────────────┐
│  Tool 列表       —— 能干的活                 │
│       ↓                                      │
│  Agent           —— 决策器（"下一步做啥"）   │
│       ↓                                      │
│  AgentExecutor   —— 执行器（"循环跑起来"）   │
└──────────────────────────────────────────────┘
```

**容易混淆**：Agent 和 AgentExecutor 是两回事！

- **Agent**：决策器，单次输出"下一步动作"。它**不会自己循环**，问它一次就只回答一次。
- **AgentExecutor**：把 Agent 套进循环里跑，处理工具调用、错误、终止条件。

实际项目中你写代码用的总是 AgentExecutor。

---

## 二、create_tool_calling_agent — 现代推荐方式 ⭐ 必练

最常用的写法，3 步搞定：

```python
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate
from langchain.agents import create_tool_calling_agent, AgentExecutor

# ============================================================
# 第 1 步：准备 LLM 和 Tools
# ============================================================
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key="your-key",
    base_url="https://api.deepseek.com",
    temperature=0,           # Agent 推荐用 0，保证稳定
)

@tool
def get_weather(city: str) -> str:
    """查询某个城市的实时天气"""
    return f"{city} 今天 25 度，晴"

@tool
def calculator(expression: str) -> str:
    """执行数学计算，例如 '2 + 3 * 4'"""
    return str(eval(expression))

tools = [get_weather, calculator]

# ============================================================
# 第 2 步:定义 Prompt 模板
# ============================================================
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个智能助手，根据需要调用工具回答用户问题。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),   # ⚠️ 这一行必须有
])

# ============================================================
# 第 3 步:组装 Agent + AgentExecutor
# ============================================================
agent = create_tool_calling_agent(llm, tools, prompt)

executor = AgentExecutor(
    agent=agent,
    tools=tools,         # tools 要传给 executor，让它知道怎么真正调用
    verbose=True,        # 打印执行过程，调试必备
)

# ============================================================
# 第 4 步:跑起来
# ============================================================
result = executor.invoke({"input": "北京天气怎么样？再帮我算 23*45 是多少"})
print(result["output"])
```

**输出**（开了 verbose）：
```
> Entering new AgentExecutor chain...

Invoking: `get_weather` with `{'city': '北京'}`
北京 今天 25 度，晴

Invoking: `calculator` with `{'expression': '23 * 45'}`
1035

北京今天 25 度，晴朗。23 × 45 = 1035。

> Finished chain.
```

可以看到 LLM 自己决定调用了两个工具，然后整合答案。

## 三、Prompt 里的 agent_scratchpad — 必须懂

```python
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是助手"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),   # ← 这是什么
])
```

`agent_scratchpad`（"草稿纸"）是 Agent 的**记事本**，保存着每一轮的思考和工具调用结果：

```
[第 1 轮] LLM 看到的内容：
  system: 你是助手
  human: 北京天气怎么样？
  agent_scratchpad: (空)
  → LLM 决定: 调用 get_weather("北京")

[第 2 轮] LLM 看到的内容：
  system: 你是助手
  human: 北京天气怎么样？
  agent_scratchpad:
    - tool_call: get_weather("北京")
    - tool_result: "北京 今天 25 度"
  → LLM 决定: 直接回答用户
```

**关键**：每次循环，AgentExecutor 都会把上一轮的工具调用和结果塞进 `agent_scratchpad`，然后再问一遍 LLM"下一步呢？"

**所以这个占位符必须留着**，否则 LLM 会失忆，反复调用同一个工具。

## 四、AgentExecutor 的常用参数

```python
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,                       # 打印中间过程
    max_iterations=10,                  # 最多循环几次（防死循环）
    max_execution_time=60,              # 最多执行多少秒
    return_intermediate_steps=True,     # 返回中间步骤（拿到每次工具调用细节）
    handle_parsing_errors=True,         # 解析错误时自动重试
)
```

### max_iterations — 最关键的"保险丝"

```python
# 不设的话，LLM 抽风可能调上百次工具，烧钱
executor = AgentExecutor(agent=agent, tools=tools, max_iterations=10)
```

如果超过限制还没结果，AgentExecutor 会强制停止，返回"无法完成"信息。

### return_intermediate_steps — 拿到调用细节

```python
executor = AgentExecutor(agent=agent, tools=tools, return_intermediate_steps=True)
result = executor.invoke({"input": "北京天气怎么样？"})

print(result["output"])              # 最终回答
print(result["intermediate_steps"])  # 中间过程
# [(AgentAction(tool='get_weather', tool_input={'city': '北京'}), '北京 25 度')]
```

**实用场景**：要在前端展示"AI 正在调用 X 工具..."这种进度提示。

## 五、create_react_agent — 老派 ReAct 方式 📖 了解即可

ReAct 不依赖 Function Calling，靠 Prompt 让 LLM 演出推理：

```python
from langchain.agents import create_react_agent, AgentExecutor
from langchain import hub

# ReAct 用一个特殊的 Prompt 模板
# 这个模板要求 LLM 输出固定格式：Thought / Action / Action Input / Observation
prompt = hub.pull("hwchase17/react")

agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

result = executor.invoke({"input": "北京天气怎么样？"})
```

**ReAct 的输出格式**：
```
Thought: 我需要查北京的天气
Action: get_weather
Action Input: 北京
Observation: 北京 今天 25 度
Thought: 我知道答案了
Final Answer: 北京今天 25 度
```

**什么时候用 ReAct**：
- 你用的 LLM 不支持 Function Calling（比如某些开源小模型）
- 需要看到 LLM 的"思考过程"做调试

**否则一律用 `create_tool_calling_agent`**。

## 六、流式输出 — 让 Agent 边想边说

很多用户场景需要流式（打字机效果），AgentExecutor 支持：

```python
# 流式输出每一步
for event in executor.stream({"input": "北京天气怎么样？"}):
    print(event)
    # 每一步事件，包括 'actions'(决定调用工具)、'steps'(工具结果)、'output'(最终回答)
```

更精细的流式（按 token 输出）需要用 `astream_events`：

```python
async for event in executor.astream_events({"input": "..."}, version="v2"):
    kind = event["event"]
    if kind == "on_chat_model_stream":
        print(event["data"]["chunk"].content, end="", flush=True)
```

## 七、错误处理 — 让 Agent 不轻易崩溃

### 1. 工具调用错误

```python
@tool
def get_weather(city: str) -> str:
    """查询天气"""
    if not city:
        return "错误：城市不能为空"   # 返回字符串而不是 raise
    return f"{city} 25 度"
```

### 2. 解析错误（ReAct 模式特有）

```python
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=True,         # 解析失败时把错误反馈给 LLM 重试
    # 也可以传一个字符串作为错误消息
    # handle_parsing_errors="格式不对，请重新输出 Final Answer"
)
```

### 3. 超时和最大迭代

```python
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=15,
    max_execution_time=30,    # 30 秒强制停止
    early_stopping_method="force",   # 强制停止（不再调 LLM）
)
```

## 八、System Prompt 怎么写

System Prompt 决定了 Agent 的"性格"和"行为准则"。常见写法：

```python
SYSTEM_PROMPT = """你是一个专业的客服助手，名字叫小 A。

你能做的事：
1. 查询订单状态（用 get_order 工具）
2. 查询商品信息（用 get_product 工具）
3. 提交售后申请（用 submit_refund 工具）

行为准则：
- 不要编造信息，不知道就调工具
- 如果用户问题与电商无关，礼貌地说明你只能处理购物相关问题
- 提交售后前一定要先查订单状态确认
- 用中文回答，语气友好
"""

prompt = ChatPromptTemplate.from_messages([
    ("system", SYSTEM_PROMPT),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
```

**写 System Prompt 的三个原则**：

```
1. 给身份：你是谁
2. 给能力清单：你能做什么、什么场景调什么工具
3. 给约束：不能干嘛、出错怎么处理
```

## 九、与上一阶段的对比

### 你之前学的 RAG Chain

```python
# 流程固定
chain = create_retrieval_chain(retriever, document_chain)
result = chain.invoke({"input": "..."})
```

适合：用户问题就是要查知识库回答。

### 现在的 Agent

```python
# 流程动态
tools = [retriever_tool, calculator, get_weather]
agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools)
result = executor.invoke({"input": "..."})
```

适合：
- 问题种类多样，需要分情况处理
- 需要多步骤（查 A 再查 B 再算）
- 有些问题该用 RAG、有些该直接回答、有些该调外部 API

## 十、调试技巧

### 1. verbose=True 是你的好朋友

```python
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
```

能看到每一步：调了什么工具、传了什么参数、得到了什么结果。

### 2. 用 LangSmith 做高级追踪 📖 了解

```python
import os
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "your-langsmith-key"
```

LangSmith 是 LangChain 官方调试平台，能可视化看每次调用的输入输出、耗时、token 数。

### 3. 工具调不出来时看 description

如果发现 Agent 总是不调用某个工具，**第一时间检查 docstring**：

```python
# ❌ 太抽象
"""查询信息的工具"""

# ✅ 给具体场景
"""查询某个城市的实时天气，包括温度、天气状况和湿度。当用户问"XX天气怎么样"时使用。"""
```

## 十一、面试怎么讲

```
面试官："你怎么实现一个客服 Agent？"

你：
"用 LangChain 的 create_tool_calling_agent。
- LLM 选了 DeepSeek，因为它支持 Function Calling 又便宜
- 工具是几个 @tool 函数：查订单、查商品、查 FAQ（这个底层是 RAG）
- Prompt 里给了客服身份、能力清单、不知道时该调哪个工具
- 用 AgentExecutor 包装，开 verbose 调试，max_iterations=10 防死循环

整个 Agent 的执行是一个循环：LLM 看用户问题决定调哪个工具，
工具结果通过 agent_scratchpad 回填给 LLM，直到 LLM 说够了输出最终回答。

最关键的工程点：每个工具的 docstring 都要写得很清楚，因为 LLM 完全
靠 description 判断要不要调、参数怎么填。"
```

## 总结

- **Agent ≠ AgentExecutor**：Agent 是决策器（一次输出），AgentExecutor 是循环执行器
- 现代写法：`create_tool_calling_agent` + `AgentExecutor`
- Prompt 必须包含 `{agent_scratchpad}` 占位符（"草稿纸"）
- AgentExecutor 关键参数：`verbose`、`max_iterations`、`return_intermediate_steps`
- `create_react_agent` 是老派方式，老模型才用
- 调试就靠 `verbose=True` 看每一步
- 下一步：让 Agent 记住对话历史（Memory，第 4 章）
