## 一、目标：做一个智能助手 Agent

设定一个真实场景：**做一个公司客服助手 Xiao A**

它能做的事：
1. **查公司知识库**（RAG，比如产品手册、政策）
2. **查订单状态**（外部 API）
3. **数学计算**（比如算折扣、优惠后价格）
4. **当前时间** （用户经常问"现在几点"）
5. **多轮对话**：记住用户身份和上下文

整个 Agent 由 **LLM + 4 个 Tool + Memory** 组成。

## 三、完整代码（边写边讲）

### Step 1: 准备知识库（RAG 部分）

复用上一阶段你学的 RAG 流程：

```python
from langchain_community.document_loaders import DirectoryLoader, TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_ollama import OllamaEmbeddings
from langchain_chroma import Chroma

def build_knowledge_base():
    """构建公司知识库（只跑一次）"""
    loader = DirectoryLoader(
        "./company_docs",
        glob="**/*.txt",
        loader_cls=TextLoader,
        loader_kwargs={"encoding": "utf-8"}
    )
    docs = loader.load()

    splitter = RecursiveCharacterTextSplitter(chunk_size=300, chunk_overlap=30)
    chunks = splitter.split_documents(docs)

    embeddings = OllamaEmbeddings(model="nomic-embed-text")
    vectorstore = Chroma.from_documents(
        documents=chunks,
        embedding=embeddings,
        persist_directory="./company_kb"
    )
    return vectorstore

vectorstore = build_knowledge_base()
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
```

### Step 2: 把 Retriever 包装成 Tool

```python
from langchain.tools.retriever import create_retriever_tool
```

### Step 3: 定义其他业务工具

```python
from langchain_core.tools import tool
from datetime import datetime

@tool
def get_order_status(order_id: str) -> str:
    """查询订单状态。

    Args:
        order_id: 订单号，例如 "ORD-12345"
    """
    # 实际项目里这里会查数据库或调内部 API
    fake_orders = {
        "ORD-12345": "已发货，预计明天送达",
        "ORD-67890": "已签收",
    }
    return fake_orders.get(order_id, f"未找到订单 {order_id}")


@tool
def calculator(expression: str) -> str:
    """执行数学计算，支持加减乘除。

    Args:
        expression: 数学表达式，例如 "200 * 0.8" 表示 200 打 8 折
    """
    try:
        return str(eval(expression))
    except Exception as e:
        return f"计算错误：{e}"


@tool
def get_current_time() -> str:
    """获取当前时间和日期"""
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")
```

### Step 5: 写 Prompt

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

SYSTEM_PROMPT = """你是公司客服助手，名字叫小 A。

【你的能力】
1. 通过 company_knowledge_base 工具查询公司产品、政策、规则
2. 通过 get_order_status 工具查询订单状态
3. 通过 calculator 工具进行数学计算（比如折扣、优惠后价格）
4. 通过 get_current_time 工具获取当前时间

【行为准则】
- 优先使用工具获取真实信息，不要编造
- 用户问"我的订单"、"我什么时候能收到"时，引导用户提供订单号
- 涉及公司业务的问题先查知识库
- 保持友好、简洁的语气
- 用中文回答
"""

prompt = ChatPromptTemplate.from_messages([
    ("system", SYSTEM_PROMPT),
    MessagesPlaceholder(variable_name="chat_history"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
```

### Step 6: 创建 Agent + Executor + Memory 包装

```python
from langchain_openai import ChatOpenAI
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_community.chat_message_histories import ChatMessageHistory

# LLM
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key="your-key",
    base_url="https://api.deepseek.com",
    temperature=0,
)

# Agent
agent = create_tool_calling_agent(llm, tools, prompt)

# Executor
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    max_iterations=10,
    handle_parsing_errors=True,
)

# Memory 包装
store = {}
def get_session_history(session_id: str):
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

agent_with_history = RunnableWithMessageHistory(
    executor,
    get_session_history,
    input_messages_key="input",
    history_messages_key="chat_history",
)
```

### Step 7: 运行

```python
config = {"configurable": {"session_id": "user_001"}}

def chat(question):
    result = agent_with_history.invoke({"input": question}, config=config)
    print(f"\n小A：{result['output']}\n")

# 测试多轮对话
chat("你好，我叫小明")
chat("我的订单 ORD-12345 现在是什么状态？")
chat("现在几点了？")
chat("你们的退款政策是什么？")
chat("一件 200 块的商品打 7 折后是多少？")
chat("我叫什么名字？")     # 测试记忆
```

### 5. 长对话裁剪

聊到几十轮后历史会撑爆，加 trim：

```python
from langchain_core.messages import trim_messages

# 在加载历史的地方加截断
def get_session_history_trimmed(session_id):
    history = store.setdefault(session_id, ChatMessageHistory())
    history.messages = trim_messages(
        history.messages,
        max_tokens=2000,
        strategy="last",
        token_counter=llm,
    )
    return history
```

## 六、流式输出（让用户能边聊边看）

```python
async def chat_stream(question, session_id):
    config = {"configurable": {"session_id": session_id}}
    async for event in agent_with_history.astream_events(
        {"input": question},
        config=config,
        version="v2",
    ):
        kind = event["event"]
        if kind == "on_chat_model_stream":
            # LLM 输出的 token
            chunk = event["data"]["chunk"]
            print(chunk.content, end="", flush=True)
        elif kind == "on_tool_start":
            # 开始调用工具
            print(f"\n[调用 {event['name']}...]")
        elif kind == "on_tool_end":
            # 工具结果
            print(f"[完成]")
```

实际接前端时，把这些事件用 SSE / WebSocket 推给浏览器即可。

## 七、Agent 工程的常见坑

| 坑 | 解决 |
|----|------|
| LLM 不调工具，直接编造答案 | docstring 写清楚使用场景；temperature=0 |
| LLM 反复调用同一个工具 | 检查 `agent_scratchpad` 占位符是否漏了 |
| 工具被调，但参数总是错的 | docstring 给参数格式示例（"YYYY-MM-DD"） |
| Agent 死循环 | 设置 `max_iterations` 兜底 |
| 工具太多 LLM 选不准 | 把工具拆成多个 Agent（"Agent of Agents"） |
| 中文 LLM 调用工具丢字段 | 升级到支持原生 tool calling 的版本（DeepSeek-V3） |
| 长对话上下文爆炸 | 用 `trim_messages` 或摘要压缩 |

## 八、整体流程回顾

```
[准备阶段]
  1. 加载文档 → 分块 → 向量化 → 存 Chroma （RAG 准备）
  2. 把 Retriever 包成 Tool
  3. 写其他业务 Tool（@tool 装饰器）
  4. 写 Prompt（system + chat_history + input + scratchpad）
  5. create_tool_calling_agent + AgentExecutor
  6. RunnableWithMessageHistory 包一层

[运行阶段]
  用户提问 → 加载历史 →
  LLM 思考 → 调工具 → 拿结果 → 再思考（循环） →
  输出回答 → 保存到历史
```

这就是一个完整的"Agentic Application"。

## 九、面试怎么讲完整 Agent

```
面试官："说说你做的 Agent 应用？"

你：
"我做了一个公司客服 Agent，技术栈是 LangChain + DeepSeek + Chroma + Redis。

【架构】
- 4 个 Tool：知识库检索（RAG 包装的）、订单查询、计算器、时间查询
- Agent 用 create_tool_calling_agent，因为 DeepSeek 支持 Function Calling
- AgentExecutor 设了 max_iterations=10 防死循环、verbose 调试
- 用 RunnableWithMessageHistory 包一层做多轮对话，历史存 Redis 按 session_id 隔离

【流程】
用户提问 → 加载该 session 历史 → LLM 决策 → 调工具/直接答 → 
更新历史 → 输出。

【关键工程点】
1. Tool 的 docstring 写得很详细，给了使用场景和参数示例
2. 把 Retriever 包成 Tool 比直接用 RAG Chain 更灵活，
   Agent 知道什么时候需要查知识库
3. 长对话用 trim_messages 截断防超 token
4. 接了 LangSmith 做调用追踪

【遇到的坑】
LLM 一开始有时候不调工具直接编答案，把 temperature 改成 0、
docstring 加上'当用户询问 XXX 时使用'，准确率上去了。"
```

## 总结

- 完整 Agent = **LLM + Tools（含 RAG 工具）+ Prompt + AgentExecutor + Memory**
- 模块化思维：每个能力都是一个 Tool，能复用、能替换
- RAG 不是独立系统，是 Agent 的一个工具
- 生产关键点：持久化历史、错误处理、限流、长对话裁剪、监控
- Agent 是 LLM 应用的"终极形态"，未来重心会从 Chain 转到 Agent + LangGraph
- 面试时要能完整讲出架构、关键决策和踩过的坑
