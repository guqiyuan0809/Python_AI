# Memory — 对话记忆

> 上一章你做出的 Agent 每次调用都是"失忆"的：用户问完一个问题，下一句它就忘了你是谁。
> 这一章解决：**怎么让 Agent 记住对话历史，做多轮问答？**

---

## 一、为什么 Agent 默认会失忆

LLM 是无状态的，每次 API 调用都是全新一次推理：

```
[第 1 轮]
User: 我叫小明
LLM: 你好小明

[第 2 轮]  ← 新的一次 API 调用
User: 我叫什么？
LLM: 我不知道你的名字   ← 因为它根本没看到第 1 轮
```

**要让 LLM"记住"，就必须每次把历史对话一起传过去**：

```
[第 2 轮]
User 实际发送给 LLM 的内容:
  - 历史: User: 我叫小明 / Assistant: 你好小明
  - 当前: User: 我叫什么？
LLM: 你叫小明
```

Memory 的本质就是：**自动管理"历史对话拼接"这件事**。

## 二、LangChain 的 Memory 演进

LangChain 的 Memory 经历了一次大改：

| 版本 | 写法 | 状态 |
|------|------|------|
| 旧版 | `ConversationBufferMemory` 等一堆类 | ⚠️ 已废弃 |
| 新版 | `RunnableWithMessageHistory` | ⭐ 现在用这个 |

旧版的 `ConversationBufferMemory` 你在网上看的教程会经常出现，但 LangChain 0.3+ 已经不推荐了。**直接学新版**。

---

## 三、新版 Memory: RunnableWithMessageHistory ⭐ 必练

### 核心思想

```
1. 准备一个"消息历史存储器"（按 session_id 区分不同用户）
2. 用 RunnableWithMessageHistory 把 Agent 包起来
3. 调用时传 session_id，框架自动加载/保存历史
```

### 完整代码

```python
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_community.chat_message_histories import ChatMessageHistory

# ============================================================
# 第 1 步：准备 LLM、Tools（和上一章一样）
# ============================================================
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key="your-key",
    base_url="https://api.deepseek.com",
)

@tool
def get_weather(city: str) -> str:
    """查询天气"""
    return f"{city} 25 度"

tools = [get_weather]

# ============================================================
# 第 2 步：Prompt 中加入历史消息占位符
# ============================================================
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个智能助手，能记住和用户的对话历史。"),
    MessagesPlaceholder(variable_name="chat_history"),    # ⚠️ 新增这一行
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# ============================================================
# 第 3 步：定义"按 session 取历史"的函数
# ============================================================
# 内存里的会话存储（实际项目可以换 Redis）
store = {}

def get_session_history(session_id: str) -> ChatMessageHistory:
    if session_id not in store:
        store[session_id] = ChatMessageHistory()
    return store[session_id]

# ============================================================
# 第 4 步:用 RunnableWithMessageHistory 包装
# ============================================================
agent_with_history = RunnableWithMessageHistory(
    executor,                                  # 被包装的 Agent
    get_session_history,                       # 怎么按 session_id 取历史
    input_messages_key="input",                # 输入字段名
    history_messages_key="chat_history",       # 历史字段名（要和 prompt 对应）
)

# ============================================================
# 第 5 步:调用（带 session_id）
# ============================================================
config = {"configurable": {"session_id": "user_001"}}

# 第 1 轮
result = agent_with_history.invoke(
    {"input": "你好，我叫小明"},
    config=config,
)
print(result["output"])   # "你好小明，..."

# 第 2 轮 —— 它能记住！
result = agent_with_history.invoke(
    {"input": "我叫什么名字？"},
    config=config,
)
print(result["output"])   # "你叫小明"
```

### 关键点解释

```python
MessagesPlaceholder(variable_name="chat_history")
```
告诉 Prompt："这里要插入一段历史对话"。

```python
get_session_history(session_id)
```
框架在每次调用时，先用这个函数加载历史，调用完再把新消息追加进去。所有这些**都是自动的**，你不用手写存取。

```python
config = {"configurable": {"session_id": "user_001"}}
```
不同的 session_id 对应不同的对话历史。多个用户同时聊天时，靠这个 ID 区分。

## 四、ChatMessageHistory — 历史的"容器"

`ChatMessageHistory` 是消息列表的简单封装：

```python
from langchain_community.chat_message_histories import ChatMessageHistory

history = ChatMessageHistory()
history.add_user_message("你好")
history.add_ai_message("你好，有什么需要帮助的吗？")

print(history.messages)
# [HumanMessage(content='你好'), AIMessage(content='你好，有什么需要帮助的吗？')]
```

`RunnableWithMessageHistory` 就是基于这个对象工作的。

## 五、持久化历史 — 不要用内存存储

上面的例子用 `store = {}` 存在内存里，**程序重启就全没了**。生产环境要用持久化存储。

### 方式 1：文件存储（学习用够了）

```python
from langchain_community.chat_message_histories import FileChatMessageHistory

def get_session_history(session_id: str):
    return FileChatMessageHistory(f"./chat_logs/{session_id}.json")
```

每个 session 存一个 JSON 文件。

### 方式 2：Redis（生产推荐）

```bash
pip install redis
```

```python
from langchain_community.chat_message_histories import RedisChatMessageHistory

def get_session_history(session_id: str):
    return RedisChatMessageHistory(
        session_id=session_id,
        url="redis://localhost:6379"
    )
```

Redis 适合多实例部署，能跨服务器共享历史。

### 方式 3：数据库（PostgreSQL/MySQL）

```python
from langchain_community.chat_message_histories import SQLChatMessageHistory

def get_session_history(session_id: str):
    return SQLChatMessageHistory(
        session_id=session_id,
        connection_string="sqlite:///chat_history.db"
    )
```

## 六、长对话的问题：上下文太长

历史会无限增长，但 LLM 有 token 限制（DeepSeek 64K，GPT-4 128K）。
聊了 100 轮后历史可能就把 prompt 撑爆了。

### 解决方案 1：trim_messages — 只保留最近 N 条

```python
from langchain_core.messages import trim_messages

trimmed = trim_messages(
    history.messages,
    max_tokens=2000,             # 最多保留 2000 tokens
    strategy="last",             # 保留最近的
    token_counter=llm,           # 用 LLM 计算 token 数
    include_system=True,         # 保留 system message
)
```

集成进 chain：

```python
from langchain_core.runnables import RunnablePassthrough

agent_with_history = RunnableWithMessageHistory(
    executor,
    get_session_history,
    input_messages_key="input",
    history_messages_key="chat_history",
).with_config(
    # 在这里加 trim 逻辑
)
```

### 解决方案 2：摘要 — 把老对话总结成一段话

```python
from langchain.memory import ConversationSummaryMemory   # 旧版还能用

# 把超过 N 条的历史让 LLM 总结成一句"用户和助手讨论了 XX 话题"
# 然后用这段摘要替代原始历史
```

实际项目中**通常是 trim + 摘要混合用**：保留最近 5 轮原文，更早的总结成一段话。

## 九、Memory 与 Tools 的协作

要注意：**Memory 只记录用户和最终回答的对话，不记录中间的工具调用**。

```
存到 Memory 里的：
  User: 北京天气怎么样？
  AI: 北京 25 度

不存的（这些只在 agent_scratchpad 里）：
  Tool call: get_weather("北京")
  Tool result: "25 度"
```

这是 LangChain 的设计选择：避免让历史塞满工具的调用日志，影响 LLM 后续决策。

## 十、与你之前学的对比

### 纯代码（你原来手动管历史）

```python
messages = [{"role": "system", "content": "你是助手"}]

while True:
    user_input = input("你: ")
    messages.append({"role": "user", "content": user_input})

    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=messages,
    )
    ai_msg = response.choices[0].message.content
    messages.append({"role": "assistant", "content": ai_msg})
    print(f"AI: {ai_msg}")
```

问题：
- 多个用户的历史要自己分开存
- 重启程序丢
- 长对话超 token 要自己截断
- 工具调用历史和对话历史的关系要自己处理

### LangChain RunnableWithMessageHistory

```python
agent_with_history = RunnableWithMessageHistory(executor, get_session_history, ...)
agent_with_history.invoke({"input": "..."}, config={"configurable": {"session_id": "001"}})
```

LangChain 帮你做了：
1. 按 session_id 自动隔离不同用户的历史
2. 自动把历史拼到 Prompt 里
3. 自动保存新对话
4. 接入 Redis/SQL 只改一个函数

## 十一、实战：给上一章的 Agent 加上记忆

把第 3 章的代码改一下：

```python
# 第 3 章原来的 Prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是助手"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

# 改成支持记忆
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是助手"),
    MessagesPlaceholder("chat_history"),    # ← 新增
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

# 包一下
agent_with_history = RunnableWithMessageHistory(
    executor,
    lambda sid: get_session_history(sid),   # 取 session 历史的函数
    input_messages_key="input",
    history_messages_key="chat_history",
)

# 调用时多传一个 session_id
agent_with_history.invoke(
    {"input": "..."},
    config={"configurable": {"session_id": "user_001"}}
)
```

**就这 2 处改动**，Agent 就有记忆了。

## 十二、面试怎么讲

```
面试官："Agent 怎么记住对话历史？"

你：
"用 LangChain 的 RunnableWithMessageHistory 包装 Agent。
核心是定义一个 get_session_history 函数：根据 session_id 返回该
用户的历史消息容器。

Prompt 里加 MessagesPlaceholder('chat_history')，框架会自动把
历史塞进去。每次调用 invoke 时传 session_id，框架自动加载历史
和保存新对话。

生产环境历史用 Redis 存，长对话用 trim_messages 截断防 token 超限。
注意 Memory 只记录对话本身，工具调用过程不会污染历史。"
```

## 总结

- LLM 默认无状态，Memory 的本质是"自动管理历史拼接"
- 新版用 `RunnableWithMessageHistory` 包装 Agent（旧版的 `ConversationBufferMemory` 已废弃）
- Prompt 必须加 `MessagesPlaceholder("chat_history")`
- 调用时传 `config={"configurable": {"session_id": "..."}}` 区分用户
- 生产用 Redis/SQL 持久化（开发用文件或内存）
- 长对话用 `trim_messages` 截断
- Memory 只记录对话，不记录中间工具调用
