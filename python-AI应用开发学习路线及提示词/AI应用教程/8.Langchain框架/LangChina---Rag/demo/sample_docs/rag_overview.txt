RAG 技术概述

RAG（Retrieval-Augmented Generation，检索增强生成）是目前最主流的 AI 应用技术方案。它通过检索外部知识库来增强大语言模型的回答能力。

RAG 的核心原理：
1. 用户提出问题
2. 系统将问题转换为向量（Embedding）
3. 在向量数据库中搜索与问题最相似的文档片段
4. 将找到的文档片段与用户问题一起发送给大语言模型
5. 大语言模型基于文档内容生成回答

RAG 解决了大语言模型的三个核心问题：
- 知识过时：模型训练数据有截止日期，RAG 可以提供最新信息
- 幻觉问题：模型可能编造不存在的信息，RAG 让回答有据可查
- 私有数据：模型不知道企业内部文档，RAG 可以接入私有知识库

RAG 的典型应用场景：
1. 企业内部知识库问答
2. 智能客服机器人
3. 技术文档助手
4. 法律法规检索
5. 医疗文献问答

RAG 的关键技术点：
- Embedding 模型选择：影响检索质量的核心因素
- 文本分块策略：chunk_size 和 overlap 的设置影响检索精度
- 向量数据库选型：Chroma（轻量）、Milvus（生产级）、FAISS（高性能）
- 检索策略：向量检索、关键词检索、混合检索
- Rerank 重排序：对检索结果二次排序，提高精度

RAG 与微调的区别：
- RAG 成本低、更新快、可解释性强
- 微调需要 GPU 训练、成本高、适合改变模型行为
- 大部分场景 RAG 就够了，特殊需求才考虑微调

RAG 的性能优化方向：
1. 混合检索：结合向量检索和 BM25 关键词检索
2. Rerank：使用重排序模型优化检索结果
3. 查询改写：对用户问题进行改写，提高检索效果
4. 分块优化：实验不同的 chunk_size，找到最佳配置
5. 引用溯源：让用户能验证回答的来源
