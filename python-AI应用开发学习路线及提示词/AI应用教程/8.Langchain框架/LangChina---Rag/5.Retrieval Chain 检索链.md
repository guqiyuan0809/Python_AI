# Retrieval Chain — 检索链

> 前面 4 步把知识库准备好了（加载 → 分块 → 向量化 → 存储），
> 这一步是最后一步：接收用户问题 → 检索相关文档 → 让 AI 基于文档生成回答。

---

## 一、Retrieval Chain 做什么

```
用户问题: "Python 有什么优点？"
      ↓
  ┌───────────┐
  │ Retriever │ → 从向量数据库检索相关文档块
  └───────────┘
      ↓
  检索到的文档:
  - "Python 语法简洁、生态丰富..."
  - "Python 被称为胶水语言..."
      ↓
  ┌───────────┐
  │    LLM    │ → 基于文档 + 问题生成回答
  └───────────┘
      ↓
  回答: "Python 的优点包括：1. 语法简洁... 2. 生态丰富... 3. ..."
```

这就是 RAG 的最后一步：**检索 + 生成**。

---

## 二、手动实现（理解原理）

在用 LangChain Chain 之前，先看看手动怎么做，和你之前学的 RAG 流程一样：

```python
from langchain_chroma import Chroma
from langchain_ollama import OllamaEmbeddings
from openai import OpenAI

# 1. 加载已有的向量库
embeddings = OllamaEmbeddings(model="nomic-embed-text")
vectorstore = Chroma(persist_directory="./chroma_db", embedding_function=embeddings)

# 2. 检索相关文档
query = "Python 有什么优点？"
docs = vectorstore.similarity_search(query, k=3)

# 3. 拼接上下文
context = "\n\n".join([doc.page_content for doc in docs])

# 4. 构造 Prompt，调用 LLM
client = OpenAI(api_key="your-key", base_url="https://api.deepseek.com")
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "根据提供的文档内容回答问题。如果文档中没有相关信息，请说明。"},
        {"role": "user", "content": f"文档内容：\n{context}\n\n问题：{query}"}
    ]
)
print(response.choices[0].message.content)
```

这就是你在 RAG 阶段学的完整流程。LangChain Chain 把这个过程封装成几行代码。

## 三、用 LangChain Chain 实现 ⭐ 必练

### 方式 1：create_retrieval_chain（推荐）

```python
from langchain_chroma import Chroma
from langchain_ollama import OllamaEmbeddings
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain.chains.retrieval import create_retrieval_chain

# 1. 初始化组件
embeddings = OllamaEmbeddings(model="nomic-embed-text")
vectorstore = Chroma(persist_directory="./chroma_db", embedding_function=embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

llm = ChatOpenAI(
    model="deepseek-chat",
    api_key="your-key",
    base_url="https://api.deepseek.com"
)

# 2. 定义 Prompt 模板
prompt = ChatPromptTemplate.from_template("""
根据以下文档内容回答问题。如果文档中没有相关信息，请说"文档中没有找到相关信息"。

文档内容：
{context}

问题：{input}
""")

# 3. 创建 Chain
# create_stuff_documents_chain：把检索到的文档"塞入"Prompt 的 {context} 位置
document_chain = create_stuff_documents_chain(llm, prompt)

# create_retrieval_chain：把 retriever 和 document_chain 串起来
#   用户问题 → retriever 检索 → 文档塞入 prompt → LLM 生成回答
chain = create_retrieval_chain(retriever, document_chain)

# 4. 运行
result = chain.invoke({"input": "Python 有什么优点？"})

print(result["answer"])            # AI 的回答
print(result["context"])           # 检索到的文档（用于引用溯源）
```

**逐行解释**：

```python
document_chain = create_stuff_documents_chain(llm, prompt)
```
这个 chain 负责：把一组文档（context）+ 用户问题（input）喂给 LLM，让 LLM 生成回答。"stuff"的意思是"把所有文档塞在一起"（最简单的策略）。

```python
chain = create_retrieval_chain(retriever, document_chain)
```
这个 chain 负责：先用 retriever 检索文档，再把文档传给 document_chain 生成回答。

```python
result = chain.invoke({"input": "Python 有什么优点？"})
```
一次调用完成整个 RAG 流程：检索 → 拼接 → 生成。

### 方式 2：LCEL 管道写法 📖 了解即可

LCEL（LangChain Expression Language）是 LangChain 的管道语法，用 `|` 符号串联步骤：

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# 格式化检索到的文档
def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

prompt = ChatPromptTemplate.from_template("""
根据以下文档内容回答问题：

文档内容：
{context}

问题：{question}
""")

# 用 | 串联：检索 → 拼接 → Prompt → LLM → 输出
chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

answer = chain.invoke("Python 有什么优点？")
print(answer)
```

**LCEL 的优势**：更灵活，可以自定义每个步骤。但对初学者来说 `create_retrieval_chain` 更直观。

## 四、引用溯源 ⭐ 重点

面试加分项。让用户知道 AI 的回答来自哪个文档的哪个部分。

### 实现方式

`create_retrieval_chain` 的返回结果里已经包含了检索到的文档：

```python
result = chain.invoke({"input": "Python 有什么优点？"})

# result["answer"] → AI 的回答
# result["context"] → 检索到的 Document 对象列表

print("回答：", result["answer"])
print("\n--- 引用来源 ---")
for doc in result["context"]:
    source = doc.metadata.get("source", "未知")
    page = doc.metadata.get("page", "")
    page_info = f"第 {page + 1} 页" if page != "" else ""
    print(f"  来源: {source} {page_info}")
    print(f"  内容: {doc.page_content[:80]}...")
    print()
```

输出效果：
```
回答：Python 的优点包括：1. 语法简洁 2. 生态丰富 3. 社区活跃...

--- 引用来源 ---
  来源: ./docs/python_guide.pdf 第 3 页
  内容: Python 是一门高级编程语言，广泛应用于 Web 开发、数据分析和人工智能...

  来源: ./docs/python_guide.pdf 第 5 页
  内容: Python 的优点包括语法简洁、生态丰富、社区活跃。它被称为"胶水语言"...
```

**引用溯源的流程回顾**：

```
Document Loaders → metadata 里自动记录 source（文件名）、page（页码）
       ↓
Text Splitters → split_documents 保留了 metadata
       ↓
Vector Stores → 存储时保留了 metadata
       ↓
Retrieval Chain → 检索结果里带着 metadata → 展示给用户
```

从第一步到最后一步，metadata 一路传递下来，这就是引用溯源的实现原理。

## 五、完整 RAG 流程代码 ⭐ 必练

把前 5 步串在一起，这就是一个完整的 LangChain RAG 系统：

```python
from langchain_community.document_loaders import DirectoryLoader, TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_ollama import OllamaEmbeddings
from langchain_chroma import Chroma
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain.chains.retrieval import create_retrieval_chain

# ============================================================
# 第 1 步：加载文档
# ============================================================
loader = DirectoryLoader("./docs", glob="**/*.txt", loader_cls=TextLoader,
                          loader_kwargs={"encoding": "utf-8"})
documents = loader.load()
print(f"加载了 {len(documents)} 个文档")

# ============================================================
# 第 2 步：分块
# ============================================================
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(documents)
print(f"分成了 {len(chunks)} 个块")

# ============================================================
# 第 3 + 4 步：向量化 + 存入向量数据库
# ============================================================
embeddings = OllamaEmbeddings(model="nomic-embed-text")
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db"
)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
print("向量数据库构建完成")

# ============================================================
# 第 5 步：创建 Retrieval Chain
# ============================================================
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key="your-key",
    base_url="https://api.deepseek.com"
)

prompt = ChatPromptTemplate.from_template("""
根据以下文档内容回答问题。如果文档中没有相关信息，请说"文档中没有找到相关信息"。

请在回答中标注信息来源。

文档内容：
{context}

问题：{input}
""")

document_chain = create_stuff_documents_chain(llm, prompt)
chain = create_retrieval_chain(retriever, document_chain)

# ============================================================
# 使用
# ============================================================
result = chain.invoke({"input": "Python 有什么优点？"})

print("\n回答：", result["answer"])
print("\n引用来源：")
for doc in result["context"]:
    print(f"  - {doc.metadata.get('source', '未知')}")
```

## 七、面试怎么讲这个流程

```
面试官："说一下你的 RAG 系统是怎么实现的？"

你：
"整个流程分 5 步：

1. 文档加载：用 LangChain 的 DirectoryLoader 批量加载 PDF 和 TXT 文件，
   自动提取文本和 metadata（文件名、页码）

2. 文本分块：用 RecursiveCharacterTextSplitter，chunk_size=500，overlap=50，
   递归分割保持语义完整

3. 向量化：用 Ollama 本地部署的 bge-m3 模型做 Embedding（学习阶段免费，
   生产可以换 OpenAI）

4. 向量存储：用 Chroma 做持久化存储，支持相似度搜索

5. 检索生成：用 create_retrieval_chain 串联检索器和 LLM，
   检索 top-3 相关文档，发给 DeepSeek 生成回答

引用溯源方面，metadata 从加载到最终输出一路传递，
回答时能标注来自哪个文件的哪一页。"
```

## 总结

- Retrieval Chain 是 RAG 流程的最后一步：检索 + 生成
- `create_retrieval_chain` 把 retriever 和 LLM 串起来，一行调用完成整个 RAG
- 引用溯源靠 metadata 一路传递：加载 → 分块 → 存储 → 检索 → 展示
- 完整流程 5 步：加载 → 分块 → 向量化+存储 → 创建 Chain → 调用
- 面试时要能完整说出这个流程，每步做了什么、为什么这么做
