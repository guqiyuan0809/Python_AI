# Vector Stores — 向量数据库

> 你在 RAG 阶段学过 Chroma 向量数据库（Demo 4），知道它能存储向量和做相似度搜索。
> 这篇讲 LangChain 怎么封装向量数据库操作，让代码更简洁。

---

## 一、LangChain 的 Vector Store 接口

LangChain 把所有向量数据库统一成同一个接口：

```python
# 不管用 Chroma、FAISS 还是 Milvus，核心方法都一样：

# 存入文档（自动向量化 + 存储）
vectorstore = Chroma.from_documents(documents=chunks, embedding=embeddings)

# 相似度搜索
results = vectorstore.similarity_search("用户问题", k=3)  #k=3 表示返回最相关的前 3 个文档块

# 转为检索器（后面接入 Chain 用）
检索器（Retriever）就是一个封装好的搜索接口，作用只有一个：


输入：用户问题（字符串）
输出：相关文档列表（Document 列表）

retriever 是 vectorstore 的"标准化包装"，让它能接入 LangChain 的各种 Chain。

retriever = vectorstore.as_retriever()
```

## 二、Chroma /ˈkroʊmə/ — 推荐使用 ⭐ 必练

你已经学过 Chroma，这里用 LangChain 的方式操作它。

### 安装

```bash
pip install langchain-chroma
```

### 基本用法：从文档创建向量库

```python
from langchain_chroma import Chroma
from langchain_ollama import OllamaEmbeddings

# 1. 初始化 Embedding 模型
embeddings = OllamaEmbeddings(model="nomic-embed-text")

# 2. 假设你已经有了分块后的 Document 列表
# chunks = splitter.split_documents(documents)

# 3. 一行代码：向量化 + 存入数据库
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
)

# 这一行代码做了三件事：
# (1) 对每个 chunk 的 page_content 调用 embeddings.embed_documents() 向量化
# (2) 把向量 + 原文 + metadata 存入 Chroma 数据库
# (3) 返回一个可以搜索的 vectorstore 对象
```

### 相似度搜索

```python
# 搜索与问题最相关的 3 个文档块
results = vectorstore.similarity_search("什么是 Python？", k=3)

for doc in results:
    print(f"来源: {doc.metadata['source']}")
    print(f"内容: {doc.page_content[:100]}...")
    print()
```

**注意**：`similarity_search` 内部自动对查询做向量化，你不需要手动调用 `embed_query`。

### 带分数的搜索
这个分数是向量之间的距离，表示两个向量"差多远
关键：Chroma 返回的是"距离"，不是"相似度"


	距离（Chroma 默认）	余弦相似度
越小	越相似 ✅	越不相似
越大	越不相似	  越相似 ✅
范围	0 ~ ∞	    -1 ~ 1


score = 0.1  → 非常相关
score = 0.5  → 比较相关  
score = 1.2  → 不太相关


```python
# 返回文档 + 相似度分数
results = vectorstore.similarity_search_with_score("什么是 Python？", k=3)

for doc, score in results:
    print(f"分数: {score:.4f} | 内容: {doc.page_content[:50]}...")
```

**分数说明**：Chroma 默认返回的是距离（越小越相似），不是余弦相似度（越大越相似）。

## 三、持久化存储 ⭐ 必练

默认情况下，Chroma 数据存在内存中，程序结束就丢失了。指定 `persist_directory` 可以持久化到磁盘：

```python
# 创建时指定存储路径
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=embeddings,
    persist_directory="./chroma_db"    # 数据保存到这个文件夹
)

# 下次启动程序，直接加载已有数据（不需要重新向量化）
vectorstore = Chroma(
    persist_directory="./chroma_db",
    embedding_function=embeddings       # 注意：加载时参数名是 embedding_function
)

# 直接搜索，数据还在
results = vectorstore.similarity_search("问题", k=3)
```

**为什么重要**：
- 文档只需要向量化一次（耗时 + 耗钱）
- 之后每次启动直接加载，不用重复处理
- 实际项目必须用持久化

## 四、as_retriever — 转为检索器 ⭐ 必练

`as_retriever()` 把 vectorstore 转成一个"检索器"对象，后面接入 Retrieval Chain 时要用：

```python
retriever = vectorstore.as_retriever(
    search_type="similarity",       # 搜索类型：相似度搜索
    search_kwargs={"k": 3}          # 返回 3 个最相关的文档
)

# retriever 可以直接调用
docs = retriever.invoke("什么是 Python？")
for doc in docs:
    print(doc.page_content[:100])
```

**search_type 选项**：

| 类型 | 说明 |
|------|------|
| `"similarity"` | 相似度搜索（默认，最常用） |
| `"mmr"` | 最大边际相关性（结果更多样化，减少重复） |

**mmr 什么时候用**：当检索结果太重复时（比如返回了 3 段内容差不多的文本），用 mmr 可以让结果更多样化。

## 五、FAISS — 另一个常用选择 📖 了解即可

FAISS 是 Facebook 开源的向量搜索库，性能好，不需要启动独立服务：

```python
from langchain_community.vectorstores import FAISS

# 用法和 Chroma 几乎一样
vectorstore = FAISS.from_documents(
    documents=chunks,
    embedding=embeddings,
)

results = vectorstore.similarity_search("问题", k=3)

# 保存到磁盘
vectorstore.save_local("./faiss_index")

# 加载
vectorstore = FAISS.load_local("./faiss_index", embeddings,
                                allow_dangerous_deserialization=True)
```

安装：
```bash
pip install faiss-cpu
```

### Chroma vs FAISS

| | Chroma | FAISS |
|--|--------|-------|
| 安装 | `pip install langchain-chroma` | `pip install faiss-cpu` |
| 持久化 | 内置支持 | 需要手动 save/load |
| metadata 过滤 | 支持 | 不支持 |
| 适用场景 | 通用，推荐 | 追求搜索速度 |

**结论**：学习和一般项目用 Chroma 就够了。

## 六、与你之前学的对比

### 纯代码（RAG 阶段 Demo 4）

```python
import chromadb

client = chromadb.Client()
collection = client.create_collection("my_collection")

# 手动添加
collection.add(
    documents=["文本1", "文本2"],
    ids=["id1", "id2"]
)

# 手动查询
results = collection.query(query_texts=["问题"], n_results=3)
```

### LangChain

```python
from langchain_chroma import Chroma

# 自动向量化 + 存储（不需要手动管理 id）
vectorstore = Chroma.from_documents(documents=chunks, embedding=embeddings)

# 搜索
results = vectorstore.similarity_search("问题", k=3)
```

**LangChain 帮你做了什么**：
1. 自动调用 Embedding 模型向量化（不用手动调）
2. 自动管理文档 ID
3. 返回的是 Document 对象（带 metadata），不是原始字典
4. `as_retriever()` 可以直接接入 Chain

## 七、完整流程（到目前为止）

```
原始文档（PDF/TXT/网页）
  ↓  Document Loaders（第 1 步）
Document 对象列表
  ↓  Text Splitters（第 2 步）
更小的 Document 块列表
  ↓  Embeddings + Vector Stores（第 3 + 4 步，自动完成）
               ↓
    Chroma.from_documents(chunks, embeddings)  ← 一行代码搞定 3+4 步
               ↓
  ↓  Retrieval Chain（第 5 步）
检索 + 生成回答
```

你会发现，Embeddings 和 Vector Stores 在 LangChain 中几乎是合在一起的。`from_documents` 这一行代码同时完成了向量化和存储。

## 总结

- Vector Store 是 RAG 流程的第四步：存储向量 + 相似度搜索
- `Chroma.from_documents(chunks, embeddings)` 一行代码完成向量化 + 存储
- `similarity_search("问题", k=3)` 搜索最相关的文档
- `persist_directory` 实现持久化，数据不丢失
- `as_retriever()` 转为检索器，接入后续的 Retrieval Chain
- 学习阶段用 Chroma 就够了
