## 一、LangChain 的 Embedding 接口

LangChain 把所有 Embedding 模型统一成同一个接口，不管你用什么模型，调用方式都一样：

```python
# 不管用哪个模型，都只有两个方法：
embeddings.embed_documents(texts)   # 批量向量化文档（列表）
embeddings.embed_query(text)        # 向量化单个查询（字符串）
```

### 为什么分两个方法？

```python
# embed_documents：一次向量化多个文档（批量，更高效）
# 用于：构建知识库时，把所有文档块一次性向量化

# embed_query：向量化一个查询
# 用于：用户提问时，把问题向量化
```

有些模型对"文档"和"查询"的向量化方式略有不同（会加不同的前缀），所以分开两个方法。

## 二、常用 Embedding 模型

### 1. OllamaEmbeddings — 本地模型 ⭐ 必练

你已经有 Ollama，用本地模型最方便（免费、无限调用）：

```python
from langchain_ollama import OllamaEmbeddings

embeddings = OllamaEmbeddings(model="nomic-embed-text")

# 向量化单个文本
vector = embeddings.embed_query("你好")
print(f"维度: {len(vector)}")   # 768

# 批量向量化
vectors = embeddings.embed_documents(["文本1", "文本2", "文本3"])
print(f"数量: {len(vectors)}")  # 3
```

安装：
```bash
pip install langchain-ollama

# 确保 Ollama 已下载 Embedding 模型
ollama pull nomic-embed-text
```

**常用本地 Embedding 模型**：

| 模型 | 维度 | 特点 | 下载命令 |
|------|------|------|---------|
| nomic-embed-text | 768 | 英文好，轻量 | `ollama pull nomic-embed-text` |
| bge-m3 | 1024 | 中英文都好，推荐 | `ollama pull bge-m3` |
| mxbai-embed-large | 1024 | 效果好，稍大 | `ollama pull mxbai-embed-large` |

**推荐**：学习阶段用 `nomic-embed-text`（小，下载快），项目中用 `bge-m3`（中文效果好）。

### 2. OpenAIEmbeddings — 云端 API 📖 了解即可

```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    api_key="your-api-key",
)

vector = embeddings.embed_query("你好")
print(f"维度: {len(vector)}")   # 1536
```

安装：
```bash
pip install langchain-openai
```

**注意**：DeepSeek 没有 Embedding API，所以用 OpenAIEmbeddings 需要 OpenAI 的 key。学习阶段直接用 Ollama 本地模型就行。

## 三、与你之前学的对比

### 纯代码（RAG 阶段）

```python
from openai import OpenAI
client = OpenAI()

# 单个文本
response = client.embeddings.create(model="text-embedding-3-small", input="你好")
vector = response.data[0].embedding

# 批量处理要自己写循环
vectors = []
for text in texts:
    response = client.embeddings.create(model="text-embedding-3-small", input=text)
    vectors.append(response.data[0].embedding)
```

### LangChain

```python
from langchain_ollama import OllamaEmbeddings
embeddings = OllamaEmbeddings(model="nomic-embed-text")

# 单个
vector = embeddings.embed_query("你好")

# 批量（一行搞定）
vectors = embeddings.embed_documents(texts)
```

**LangChain 帮你做了什么**：
1. 统一接口：换模型只改一行初始化代码，其他不用动
2. 自动批量处理：embed_documents 内部自动做批处理优化
3. 与 Vector Store 无缝对接：后面存入向量数据库时，不需要手动调用 Embedding

---

## 四、重要：后续你几乎不需要手动调用 Embedding

在 LangChain 的完整 RAG 流程中，Embedding 是**自动完成**的：

```python
# 存入向量数据库时，自动调用 embeddings 对文档向量化
vectorstore = Chroma.from_documents(documents=chunks, embedding=embeddings)

# 检索时，自动对查询向量化
results = vectorstore.similarity_search("用户问题")
```

你不需要手动调用 `embed_query` 或 `embed_documents`，向量数据库会自动帮你调用。

所以 Embedding 这一层，你要理解的是：
1. **原理**：你在 RAG 阶段已经学透了（余弦相似度、语义搜索）
2. **选型**：学习用 Ollama 本地模型，生产用云端 API
3. **使用**：初始化后传给 Vector Store，后续自动调用

## 五、完整流程（到目前为止）

```
原始文档（PDF/TXT/网页）
  ↓  Document Loaders（第 1 步）
Document 对象列表
  ↓  Text Splitters（第 2 步）
更小的 Document 块列表
  ↓  Embeddings（第 3 步）← 你现在在这里
向量（数字列表）
  ↓  Vector Stores（第 4 步）
存入向量数据库
  ↓  Retrieval Chain（第 5 步）
检索 + 生成回答
```

## 总结

- LangChain 统一了 Embedding 接口：`embed_query`（单个）和 `embed_documents`（批量）
- 学习阶段用 OllamaEmbeddings + 本地模型（免费）
- 实际项目中 Embedding 是自动调用的，你只需要初始化后传给 Vector Store
- Embedding 的原理你已经学透了，这里只是换了个调用方式
