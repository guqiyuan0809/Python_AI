# Document Loaders — 文档加载

> 你在 RAG 阶段学过文档处理 Pipeline 的第一步：加载文档。
> 当时是手动写代码读文件，现在用 LangChain 的 Document Loaders，一行代码搞定。

---

## 一、Document Loaders 是什么

LangChain 提供了大量的文档加载器，能从各种来源加载数据：

```
TextLoader        → 纯文本文件（.txt）
PyPDFLoader       → PDF 文件
Docx2txtLoader    → Word 文件（.docx）
CSVLoader         → CSV 表格
WebBaseLoader     → 网页
DirectoryLoader   → 一次性加载整个文件夹
```

**核心价值**：不用自己写文件读取逻辑，LangChain 帮你处理不同格式的差异。

---

## 二、Document 对象

所有加载器返回的都是 `Document` 对象列表，结构非常简单：

```python
from langchain_core.documents import Document

doc = Document(
    page_content="这是文档的文本内容...",   # 文本内容（字符串）
    metadata={"source": "readme.txt"}       # 元数据（字典）
)
```

**只有两个字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| page_content | str | 文档的文本内容 |
| metadata | dict | 附加信息（来源文件名、页码等） |

**metadata 的作用**：做引用溯源时，能告诉用户"这段回答来自哪个文件的第几页"。

---

## 三、常用加载器

### 1. TextLoader — 加载文本文件 ⭐ 必练

最基础的加载器，加载 .txt 文件：

```python
from langchain_community.document_loaders import TextLoader

loader = TextLoader("./docs/readme.txt", encoding="utf-8")
documents = loader.load()

print(len(documents))                    # 1（一个文件 = 一个 Document）
print(documents[0].page_content[:100])   # 文本内容前 100 字
print(documents[0].metadata)             # {'source': './docs/readme.txt'}
```

**注意**：中文文件一定要加 `encoding="utf-8"`，否则可能乱码。

---

### 2. PyPDFLoader — 加载s 必练

实际项目中最常用的格式，企业知识库大量 PDFs：

```python
from langchain_community.document_loaders import PyPDFLoader

loader = PyPDFLoader("./docs/report.pdf")
documents = loader.load()

# PDF 按页分割，每页是一个 Document
print(len(documents))                    # 页数
print(documents[0].page_content[:100])   # 第 1 页内容
print(documents[0].metadata)             # {'source': './docs/report.pdf', 'page': 0}
```

**关键点**：PDF 自动按页分割，metadata 里有 `page` 字段，做引用溯源时可以精确到第几页。

安装依赖：
```bash
pip install pypdf
```

---

### 3. WebBaseLoader — 加载网页 📖 了解即可

从网页 URL 加载内容：

```python
from langchain_community.document_loaders import WebBaseLoader

# 直接给 URL 就行，LangChain 内部自动：
# 1. 发送 HTTP 请求获取网页
# 2. 用 BeautifulSoup 解析 HTML
# 3. 去除标签，提取纯文本
# 4. 返回 Document 对象
loader = WebBaseLoader("https://docs.python.org/3/tutorial/index.html")
documents = loader.load()

print(documents[0].page_content[:200])   # 网页文本内容（HTML 标签已去除）
print(documents[0].metadata)             # {'source': 'https://...', 'title': '...'}
```

**注意**：有些网页有反爬虫限制，加载会失败（比如需要登录的页面）。

安装依赖：
```bash
pip install beautifulsoup4
```

---

### 4. CSVLoader — 加载 CSV 表格 📖 了解即可

```python
from langchain_community.document_loaders import CSVLoader

loader = CSVLoader("./docs/data.csv", encoding="utf-8")
documents = loader.load()

# 每行是一个 Document
print(len(documents))                    # 行数
print(documents[0].page_content)         # "列名1: 值1\n列名2: 值2\n..."
```

---

### 5. DirectoryLoader — 加载整个文件夹 ⭐ 必练

一次性加载文件夹下所有文件：

```python
from langchain_community.document_loaders import DirectoryLoader, TextLoader

# 加载 docs 文件夹下所有 .txt 文件
loader = DirectoryLoader(
    "./docs",
    glob="**/*.txt",               # 匹配模式：所有 .txt 文件
    loader_cls=TextLoader,          # 用 TextLoader 加载每个文件
    loader_kwargs={"encoding": "utf-8"}
)
documents = loader.load()

print(f"共加载 {len(documents)} 个文件")
for doc in documents:
    print(f"  来源: {doc.metadata['source']}")
```

**glob 参数说明**：
- `"**/*.txt"` — 所有子目录下的 .txt 文件
- `"*.pdf"` — 当前目录下的 .pdf 文件
- `"**/*.*"` — 所有文件

## 四、加载器汇总

| 加载器 | 用途 | 安装依赖 | 优先级 |
|--------|------|---------|--------|
| TextLoader | 纯文本 | 无 | ⭐ 必练 |
| PyPDFLoader | PDF | `pip install pypdf` | ⭐ 必练 |
| DirectoryLoader | 整个文件夹 | 无 | ⭐ 必练 |
| CSVLoader | CSV 表格 | 无 | 📖 了解 |
| WebBaseLoader | 网页 | `pip install beautifulsoup4` | 📖 了解 |
| Docx2txtLoader | Word | `pip install docx2txt` | 📖 了解 |
| UnstructuredMarkdownLoader | Markdown | `pip install unstructured` | ⏭️ 需要时再学 |

## 五、与你之前学的对比

### 纯代码（RAG 阶段）

```python
# 自己写文件读取
with open("readme.txt", "r", encoding="utf-8") as f:
    text = f.read()
# 没有 metadata，没有统一接口
```

### LangChain

```python
# 一行代码，自动处理格式差异
loader = TextLoader("readme.txt", encoding="utf-8")
documents = loader.load()
# 有 metadata（来源文件名），统一的 Document 接口
```

**LangChain 帮你做了什么**：
1. 统一的 Document 对象（page_content + metadata）
2. 自动处理不同文件格式的差异
3. metadata 自动记录来源信息，为引用溯源打基础
4. DirectoryLoader 批量加载，不用自己写循环

## 六、实际项目中的用法

一个典型的知识库，可能同时包含 PDF、TXT、Markdown 等格式：

```python
from langchain_community.document_loaders import (
    DirectoryLoader, TextLoader, PyPDFLoader
)

# 分别加载不同格式
txt_loader = DirectoryLoader("./docs", glob="**/*.txt", loader_cls=TextLoader,
                              loader_kwargs={"encoding": "utf-8"})
pdf_loader = DirectoryLoader("./docs", glob="**/*.pdf", loader_cls=PyPDFLoader)

# 合并所有文档
all_documents = txt_loader.load() + pdf_loader.load()
print(f"共加载 {len(all_documents)} 个文档")
```

加载完成后，下一步就是**文本分块**（Text Splitters）。

---

## 总结

- Document Loaders 是 LangChain RAG 流程的第一步：加载原始文档
- 所有加载器返回统一的 `Document` 对象（page_content + metadata）
- metadata 记录来源信息，是实现引用溯源的基础
- 最常用的三个：TextLoader、PyPDFLoader、DirectoryLoader
- 加载完成后，进入下一步：Text Splitters 文本分块
