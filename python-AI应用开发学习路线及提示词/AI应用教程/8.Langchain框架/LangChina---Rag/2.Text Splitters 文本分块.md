# Text Splitters — 文本分块

> 你在 RAG 阶段学过文本分块的原理（chunk_size=500, overlap=50），也提到过"实际项目中用 LangChain 的 RecursiveCharacterTextSplitter"。
> 这篇就讲 LangChain 的分块工具怎么用。

---

## 一、为什么要分块

你已经知道原因，简单回顾：

```
一整篇 10000 字的文档
  ↓ 分块
[块1: 500字] [块2: 500字] [块3: 500字] ...

好处：
1. 检索更精准（小块更容易匹配用户问题）
2. 节省 Token（只把相关的块发给 AI，不是整篇文档）
3. Embedding 模型有长度限制
```

---

## 二、LangChain 的分块器

### 核心类：RecursiveCharacterTextSplitter ⭐ 必练

这是 LangChain 最常用、最推荐的分块器：

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,        # 每块最大 500 字符
    chunk_overlap=50,      # 相邻块重叠 50 字符

    #分隔符列表，按照顺序依次分隔
    separators=["\n\n", "\n", "。", "！", "？", ".", " ", ""]
    # 分隔符列表，按优先级从高到低依次尝试：
    # "\n\n" → 段落（两个换行，最优先，语义最完整）
    # "\n"   → 换行（单个换行，按行分）
    # "。"   → 中文句号
    # "！"   → 中文感叹号
    # "？"   → 中文问号
    # "."    → 英文句号
    # " "    → 空格（按词分）
    # ""     → 按字符分（最后兜底，语义最差）
    # 默认值是 ["\n\n", "\n", " ", ""]，中文项目建议加上句号等中文标点
)
```

**"递归"是什么意思？**

它按 separators 列表从左到右尝试分割：

```
第 1 优先：按段落分（\n\n）
  ↓ 如果分出来的块还是太大
第 2 优先：按换行分（\n）
  ↓ 还是太大
第 3 优先：按句号分（。）
  ↓ 还是太大
第 4 优先：按空格分（ ）
  ↓ 还是太大
最后兜底：按字符分（""）
```

**核心思想**：尽量保持语义完整。先按大单位分，分不开再按小单位分。

### 基本用法

#### 1. 分割纯文本

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text = """
Python 是一门高级编程语言，广泛应用于 Web 开发、数据分析和人工智能等领域。

Python 的优点包括语法简洁、生态丰富、社区活跃。它被称为"胶水语言"，能与其他语言无缝集成。

FastAPI 是一个基于 Python 的现代 Web 框架。它支持异步操作，性能优秀，并且能自动生成 API 文档。
"""

splitter = RecursiveCharacterTextSplitter(
    chunk_size=100,       # 为了演示效果，设小一点
    chunk_overlap=20,     # 相邻两个块之间重叠的字符数
)

chunks = splitter.split_text(text)

for i, chunk in enumerate(chunks):
    print(f"--- 块 {i+1}（{len(chunk)} 字符）---")
    print(chunk)
    print()
```

#### 2. 分割 Document 对象 ⭐ 必练

实际项目中，文档加载后得到的是 Document 对象，用 `split_documents` 方法：

```python
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 第 1 步：加载文档
loader = TextLoader("./docs/readme.txt", encoding="utf-8")
documents = loader.load()

# 第 2 步：分块
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
)
chunks = splitter.split_documents(documents)

print(f"原始文档数: {len(documents)}")
print(f"分块后数量: {len(chunks)}")

# 每个 chunk 仍然是 Document 对象，保留了 metadata
for chunk in chunks[:3]:
    print(f"来源: {chunk.metadata['source']}, 长度: {len(chunk.page_content)}")
```

**关键点**：`split_documents` 会保留每个块的 metadata，这样做引用溯源时能知道每个块来自哪个文件。

## 三、两个核心参数

### chunk_size — 每块最大字符数

```python
# 太小：语义不完整，检索效果差
chunk_size=100   # "Python 是一门高级编程语言，广泛应用于" ← 句子被截断

# 太大：检索不精准，Token 浪费
chunk_size=2000  # 一大段文本，可能包含多个不相关的话题

# 推荐范围
chunk_size=300~800  # 实际项目中最常用的范围
```

### chunk_overlap — 相邻块重叠字符数

```
chunk_overlap=0 时：
[块1: ...语言广泛应用于] [块2: Web 开发数据分析...]
                       ↑ 语义在边界被切断

chunk_overlap=50 时：
[块1: ...语言广泛应用于 Web 开发] 
              [块2: 广泛应用于 Web 开发数据分析...]
              ↑ 重叠部分，边界信息不会丢失
```

**推荐配置**：`chunk_overlap` = `chunk_size` 的 10%~20%

## 四、其他分块器

### CharacterTextSplitter 📖 了解即可

只按一个分隔符分割（不递归）：

```python
from langchain_text_splitters import CharacterTextSplitter

splitter = CharacterTextSplitter(
    separator="\n\n",     # 只按段落分
    chunk_size=500,
    chunk_overlap=50,
)
```

和 RecursiveCharacterTextSplitter 的区别：
- CharacterTextSplitter：只用一个分隔符，简单粗暴
- RecursiveCharacterTextSplitter：多个分隔符依次尝试，更智能

**结论**：直接用 RecursiveCharacterTextSplitter 就行。

## 五、分块策略选择

| 场景 | 推荐配置 |
|------|---------|
| 通用文档 | chunk_size=500, overlap=50 |
| 技术文档（代码多） | chunk_size=800, overlap=100 |
| 短文本（FAQ） | chunk_size=200, overlap=30 |
| 法律/合同文档 | chunk_size=1000, overlap=150 |

**没有"最优"配置**，需要根据实际效果调整。面试时说"我测试了不同的 chunk_size，最终选了 500 是因为..."比背一个数字强得多。

## 六、与你之前学的对比

### 纯代码（RAG 阶段 Demo 5）

```python
# 手动实现重叠分块
def split_text(text, chunk_size=500, overlap=50):
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start = end - overlap
    return chunks
```

问题：按字符数硬切，可能切断句子和段落。

### LangChain

```python
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(documents)
```

优势：
1. 递归分割，尽量保持语义完整
2. 自动保留 metadata
3. 一行代码搞定

## 七、完整流程（到目前为止）

```
原始文档（PDF/TXT/网页）
  ↓  Document Loaders（第 1 步）
Document 对象列表 [page_content + metadata]
  ↓  Text Splitters（第 2 步）← 你现在在这里
更小的 Document 块列表
  ↓  Embeddings（第 3 步）
向量化
  ↓  Vector Stores（第 4 步）
存入向量数据库
  ↓  Retrieval Chain（第 5 步）
检索 + 生成回答
```

## 总结

- Text Splitters 是 RAG 流程的第二步：把大文档分成小块
- **RecursiveCharacterTextSplitter** 是最常用的分块器，递归尝试多种分隔符
- 两个核心参数：chunk_size（块大小）和 chunk_overlap（重叠大小）
- `split_documents` 方法会保留 metadata，为引用溯源打基础
- 推荐起步配置：chunk_size=500, chunk_overlap=50
- 实际项目中需要根据效果调整参数，面试时要能说出"为什么选这个值"
