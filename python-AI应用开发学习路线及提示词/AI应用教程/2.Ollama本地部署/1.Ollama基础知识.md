# Ollama 基础知识

> Ollama 是本地模型管理工具，类似 Docker 管理容器的方式来管理和运行大模型。
>
> **核心要点：Ollama 的使用和 Skills 阶段的云端 API 调用完全一样，只是一个在本地跑模型，一个在远程云端跑模型。代码只需要改 `base_url` 和 `model`，其他一字不改。**

---

## 一、Ollama 是什么

Ollama 帮你做三件事：
1. **下载模型** — 一行命令下载开源模型
2. **运行模型** — 本地启动，直接对话
3. **提供 API 服务** — 兼容 OpenAI 格式，端口 11434

---

## 二、云端 API vs 本地部署

| | 云端 API（DeepSeek/OpenAI） | 本地部署（Ollama） |
|--|--|--|
| 模型在哪跑 | 厂商的服务器 | 你自己的电脑 |
| 费用 | 按 Token 收费 | 免费，无限调用 |
| 网络 | 需要联网 | 不需要 |
| 隐私 | 数据发到第三方服务器 | 数据不出本地 |
| 模型质量 | GPT-4、Claude 等顶级模型 | 开源模型，能力稍弱 |
| 速度 | 取决于网络 | 取决于电脑配置（CPU/GPU/内存） |

---

## 三、本地部署的价值

- 开发调试时免费无限调用，不用担心 Token 费用
- 数据敏感场景必须用本地模型（企业内部数据、隐私信息）
- 学习 RAG 和 Agent 时可以用本地模型省钱
- 不依赖网络，断网也能用

---

## 四、硬件要求

本地模型对电脑配置有要求，主要看内存和显卡：

| 内存 | 能跑的模型大小 | 体验 |
|------|--------------|------|
| 8G | 1.5B - 3B | 能用，能力有限 |
| 16G | 3B - 7B | 够用，日常开发调试没问题 |
| 32G | 7B - 14B | 效果不错 |

| 显卡 | 说明 |
|------|------|
| NVIDIA 独显（6G+显存） | 最佳，GPU 加速，速度快 |
| 集成显卡 | 用 CPU 跑，速度慢但能用 |
| 无显卡 | 纯 CPU 跑，速度最慢 |

**模型大小说明**：1.5B = 15亿参数，7B = 70亿参数，参数越多能力越强但越吃配置。

## 五、Ollama 支持的常见开源模型

| 模型 | 厂商 | 可选大小 | 特点 |
|------|------|---------|------|
| Qwen2.5 | 阿里 | 0.5B / 1.5B / 3B / 7B / 14B / 32B | 中文能力好，推荐 |
| Llama3 | Meta | 8B / 70B | 英文强，社区生态最好 |
| DeepSeek | 深度求索 | 1.5B / 7B / 16B | 代码能力强 |
| Phi-3 | 微软 | 3.8B / 14B | 小模型里能力不错 |
| Gemma2 | Google | 2B / 9B / 27B | 综合能力好 |

只要是 Ollama 模型库（https://ollama.com/library）里有的，一行命令就能下载运行。

不在库里的开源模型需要自己转换格式（GGUF），比较麻烦，一般不需要。

## 六、常用命令

```bash
# 下载模型
ollama pull qwen2.5:3b

# 运行模型（进入对话模式）
ollama run qwen2.5:3b

# 查看已下载的模型
ollama list

# 删除模型（释放磁盘空间）
ollama rm qwen2.5:3b

# 查看正在运行的模型
ollama ps

# 停止模型
ollama stop qwen2.5:3b
```

## 七、API 调用（重点）

Ollama 启动后会在本地开一个 API 服务，地址是 `http://localhost:11434`，兼容 OpenAI 格式。

也就是说，你之前学的 API 调用代码，只需要改 `base_url`，就能调用本地模型：

```python
from openai import OpenAI

# 云端 API
client = OpenAI(api_key="sk-xxx", base_url="https://api.deepseek.com")

# 本地 Ollama —— 只改了 base_url 和 api_key
client = OpenAI(api_key="ollama", base_url="http://localhost:11434/v1")

# 调用方式完全一样
response = client.chat.completions.create(
    model="qwen2.5:3b",  # 改成本地模型名
    messages=[{"role": "user", "content": "你好"}]
)
print(response.choices[0].message.content)
```

**一套代码，改两行参数，就能在云端和本地之间切换。这就是 OpenAI 格式兼容的好处。**

## 常见坑

### 1. 调用本地 Ollama 时报 502 错误

**原因一：模型冷启动**。第一次调用时模型需要加载到内存，等几秒再跑一次即可。

**原因二：开了代理（梯子）**。代理会把 `localhost:11434` 的请求转发到外部，导致连接失败。**解决方法：关闭代理/梯子后再调用本地 Ollama。**

## 九、实际开发中的使用建议

1. **开发调试用本地模型**（免费），上线前用云端好模型测试效果
2. **代码里做好抽象**，base_url 和 model 用配置文件管理，方便切换
3. **本地模型适合**：学习练手、开发调试、数据敏感场景、离线环境
4. **云端模型适合**：对质量要求高的生产环境、需要顶级模型能力的场景
