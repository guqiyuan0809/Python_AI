# Embedding — 文本向量化

## 什么是 Embedding？

Embedding 就是把文字转换成数字列表（向量），让计算机能够"理解"和"计算"文字的相似度。

### 直观理解

```
"你好" → [0.2, 0.8, 0.1, 0.5, ...]  (实际有几百到几千维)
"你好吗" → [0.3, 0.7, 0.2, 0.4, ...]
"天气真好" → [0.1, 0.2, 0.9, 0.3, ...]
```

**核心原理**：意思相近的文字，向量也相近。

### 为什么需要 Embedding？

传统的文字匹配：
```python
"你好" == "你好吗"  # False，完全不匹配
"Python 教程" == "Python 学习指南"  # False
```

使用 Embedding 后：
```python
similarity("你好", "你好吗")  # 0.85（很相似）
similarity("Python 教程", "Python 学习指南")  # 0.92（非常相似）
similarity("你好", "天气")  # 0.12（不相似）
```

这样就能实现**语义搜索**，而不是简单的关键词匹配。

## Embedding 的工作原理

### 1. 向量空间

想象一个多维空间，每个词都是空间中的一个点：

```
        相似的词聚在一起
        
    你好 ●
         ● 你好吗
    早上好 ●
    
                    ● Python
                    ● 编程
                    ● 代码
                    
        ● 天气
        ● 晴天
```

### 2. 相似度计算 ⭐ 必练

最常用的是**余弦相似度**（Cosine Similarity）：

```python
import numpy as np

def cosine_similarity(vec1, vec2):
    """计算两个向量的余弦相似度"""
    dot_product = np.dot(vec1, vec2)
    norm1 = np.linalg.norm(vec1)
    norm2 = np.linalg.norm(vec2)
    return dot_product / (norm1 * norm2)

# 示例
vec1 = [0.2, 0.8, 0.1]
vec2 = [0.3, 0.7, 0.2]
similarity = cosine_similarity(vec1, vec2)
print(f"相似度: {similarity}")  # 0.98（非常相似）
```

**为什么必练**：理解相似度计算是 RAG 检索的核心原理。

**相似度范围**：
- 1.0：完全相同
- 0.8-1.0：非常相似
- 0.5-0.8：有些相似
- 0.0-0.5：不太相似
- 0.0：完全不相关

### 3. 向量维度

常见的 Embedding 模型维度：
- OpenAI `text-embedding-3-small`：1536 维
- OpenAI `text-embedding-3-large`：3072 维
- 开源模型 `bge-large-zh`：1024 维

**维度越高**：
- 表达能力越强，能捕捉更细微的语义差异
- 但存储和计算成本也越高

## Embedding 的实际应用

### 应用 1：语义搜索 ⭐ 必练

```python
from openai import OpenAI
import numpy as np

client = OpenAI()

# 知识库文档
documents = [
    "Python 是一门高级编程语言",
    "FastAPI 是一个现代化的 Web 框架",
    "机器学习是人工智能的一个分支",
    "深度学习使用神经网络"
]

# 1. 对所有文档向量化
doc_vectors = []
for doc in documents:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=doc
    )
    doc_vectors.append(response.data[0].embedding)

# 2. 用户提问
query = "如何学习 Python Web 开发？"
query_response = client.embeddings.create(
    model="text-embedding-3-small",
    input=query
)
query_vector = query_response.data[0].embedding

# 3. 计算相似度，找最相关的文档
def cosine_similarity(vec1, vec2):
    return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))

similarities = []
for i, doc_vec in enumerate(doc_vectors):
    sim = cosine_similarity(query_vector, doc_vec)
    similarities.append((i, sim))

# 4. 按相似度排序
similarities.sort(key=lambda x: x[1], reverse=True)

# 5. 输出最相关的文档
print("最相关的文档：")
for i, sim in similarities[:2]:
    print(f"相似度 {sim:.3f}: {documents[i]}")
```

输出：
```
最相关的文档：
相似度 0.856: FastAPI 是一个现代化的 Web 框架
相似度 0.782: Python 是一门高级编程语言
```

**为什么必练**：这就是 RAG 检索的完整流程，理解这个就理解了 RAG 的核心。

```python
    # text = text.lower()

text = "  Python   是一门  编程语言  "

```

### 2. 长文本处理

Embedding 模型通常有最大长度限制（如 8192 tokens）。

**策略 1：截断**
```python
def truncate_text(text, max_tokens=8000):
    """截断文本"""
    # 简单估算：1 个中文字 ≈ 1 token
    if len(text) > max_tokens:
        return text[:max_tokens]
    return text
```

**策略 2：分块后平均**
```python
def embed_long_text(text, chunk_size=8000):
    """长文本分块向量化，然后取平均"""
    chunks = [text[i:i+chunk_size] for i in range(0, len(text), chunk_size)]
    
    vectors = []
    for chunk in chunks:
        vec = embed_text(chunk)
        vectors.append(vec)
    
    # 取平均向量
    avg_vector = np.mean(vectors, axis=0)
    return avg_vector
```

### 3. 缓存机制

相同的文本不需要重复向量化：

```python
import hashlib
import pickle

class EmbeddingCache:
    def __init__(self, cache_file="embedding_cache.pkl"):
        self.cache_file = cache_file
        self.cache = self._load_cache()
    
    def _load_cache(self):
        try:
            with open(self.cache_file, 'rb') as f:
                return pickle.load(f)
        except:
            return {}
    
    def _save_cache(self):
        with open(self.cache_file, 'wb') as f:
            pickle.dump(self.cache, f)
    
    def get_embedding(self, text):
        # 计算文本的哈希值作为 key
        text_hash = hashlib.md5(text.encode()).hexdigest()
        
        if text_hash in self.cache:
            return self.cache[text_hash]
        
        # 调用 API 获取向量
        vector = embed_text(text)
        
        # 缓存
        self.cache[text_hash] = vector
        self._save_cache()
        
        return vector

# 使用
cache = EmbeddingCache()
vec1 = cache.get_embedding("你好")  # 调用 API
vec2 = cache.get_embedding("你好")  # 从缓存读取，不调用 API
```

## 常见问题

### Q1：Embedding 需要训练吗？

**不需要**。Embedding 模型已经训练好了，你直接调用 API 或加载模型就行。

### Q2：不同模型的向量能混用吗？

**不能**。不同模型生成的向量维度和语义空间不同，不能直接比较。

```python
# ❌ 错误：混用不同模型
vec1 = openai_embed("你好")  # OpenAI 模型
vec2 = bge_embed("你好吗")   # BGE 模型
similarity(vec1, vec2)  # 结果没有意义
```

### Q3：中文和英文能一起向量化吗？

**可以**，但要选择支持多语言的模型：
- OpenAI 的模型支持多语言
- `bge-m3` 支持多语言
- `bge-large-zh` 只支持中文

### Q4：向量化后能还原成文本吗？

**不能**。向量化是单向的，无法从向量还原出原始文本。

## 实战练习

### 练习 1：实现一个简单的语义搜索

```python
from openai import OpenAI
import numpy as np

client = OpenAI()

def semantic_search(query, documents, top_k=3):
    """语义搜索"""
    # 1. 向量化所有文档
    doc_response = client.embeddings.create(
        model="text-embedding-3-small",
        input=documents
    )
    doc_vectors = [item.embedding for item in doc_response.data]
    
    # 2. 向量化查询
    query_response = client.embeddings.create(
        model="text-embedding-3-small",
        input=query
    )
    query_vector = query_response.data[0].embedding
    
    # 3. 计算相似度
    similarities = []
    for i, doc_vec in enumerate(doc_vectors):
        sim = np.dot(query_vector, doc_vec) / (
            np.linalg.norm(query_vector) * np.linalg.norm(doc_vec)
        )
        similarities.append((i, sim))
    
    # 4. 排序并返回 top_k
    similarities.sort(key=lambda x: x[1], reverse=True)
    return [(documents[i], sim) for i, sim in similarities[:top_k]]

# 测试
docs = [
    "Python 是一门编程语言",
    "FastAPI 用于构建 API",
    "机器学习是 AI 的分支",
    "深度学习使用神经网络"
]

results = semantic_search("如何学习 Web 开发？", docs)
for doc, sim in results:
    print(f"{sim:.3f}: {doc}")
```

### 练习 2：实现文档去重

```python
def remove_duplicates(documents, threshold=0.9):
    """去除重复文档"""
    vectors = embed_documents(documents)
    
    keep = [True] * len(documents)
    
    for i in range(len(vectors)):
        if not keep[i]:
            continue
        for j in range(i+1, len(vectors)):
            if not keep[j]:
                continue
            sim = cosine_similarity(vectors[i], vectors[j])
            if sim > threshold:
                keep[j] = False  # 标记为重复
    
    return [doc for i, doc in enumerate(documents) if keep[i]]

# 测试

    "Python 是一种编程语言",  # 重复
    "FastAPI 是 Web 框架",
]

unique_docs = remove_duplicates(docs)
print(f"去重后剩余 {len(unique_docs)} 个文档")
```

## 深入理解：为什么要计算相似度？

### 相似度计算的本质

RAG 的核心就是"找到相关的文档"，而相似度计算就是用来判断"哪些文档和用户的问题最相关"。

**实际场景举例**：

假设你的知识库里有 1000 篇文档，用户问了一个问题：

```
用户问题："如何用 Python 调用 OpenAI API？"
```

你的知识库里有这些文档：
- 文档 A：《OpenAI SDK 详解》
- 文档 B：《Ollama 本地部署》
- 文档 C：《RAG 核心概念》
- 文档 D：《提示词类型》
- ... 还有 996 篇

RAG 系统需要做的事：
1. 把用户问题转成向量：`[0.2, 0.8, 0.1, ...]`（1536 维）
2. 把每篇文档也转成向量（之前已经存在向量数据库里了）
3. **计算用户问题向量和每篇文档向量的相似度**
4. 找出相似度最高的 3-5 篇文档
5. 把这些文档和用户问题一起发给 AI，让 AI 基于这些文档回答

### 相似度计算的作用

```python
用户问题向量 = [0.2, 0.8, 0.1, ...]

文档 A 向量 = [0.25, 0.75, 0.15, ...]  → 相似度 0.98（非常相关！）
文档 B 向量 = [0.1, 0.3, 0.9, ...]    → 相似度 0.45（不太相关）
文档 C 向量 = [0.5, 0.2, 0.7, ...]    → 相似度 0.32（不相关）
文档 D 向量 = [0.15, 0.6, 0.2, ...]   → 相似度 0.85（比较相关）

排序后：文档 A（0.98）> 文档 D（0.85）> 文档 B（0.45）> ...

最终选择：把文档 A 和文档 D 发给 AI
```

### 为什么用余弦相似度？

余弦相似度公式确实是固定的，但它有个重要特性：**只看方向，不看长度**

```python
# 余弦相似度只看方向，不看长度
vec1 = [1, 2, 3]
vec2 = [2, 4, 6]  # vec2 是 vec1 的 2 倍

# 余弦相似度 = 1.0（完全相同方向）
# 即使数值不同，但意思相同，相似度就高
```

**方向相似的含义**：

```python
# 在二维平面上（实际是 1536 维，但原理一样）
vec1 = [1, 2, 3]    # 方向：往右上方
vec2 = [2, 4, 6]    # 方向：也是往右上方（只是走得更远）
vec3 = [-1, -2, -3] # 方向：往左下方（完全相反）

cosine_similarity(vec1, vec2) = 1.0   # 方向完全一致
cosine_similarity(vec1, vec3) = -1.0  # 方向完全相反
```

这对文本很合适，因为：
- "Python API 调用" 和 "如何用 Python 调用 API" 意思一样，只是长度不同
- 余弦相似度会认为它们非常相似（接近 1.0）

### 实际文本例子

```python
# Embedding 模型会把语义相似的文本转成方向相似的向量

"如何调用 OpenAI API"     → [0.2, 0.8, 0.1, ...]  # 方向 A
"OpenAI API 调用方法"     → [0.25, 0.75, 0.15, ...] # 方向 A'（很接近）
"Python 快速排序算法"     → [0.7, 0.1, 0.5, ...]  # 方向 B（完全不同）

相似度：
- 前两个：0.98（方向几乎一样，语义相似）
- 第一个和第三个：0.32（方向差很多，语义不相关）
```

### 关键点总结

1. **相似度 = 找到和问题相关的文档**
   - 用户问："如何调用 OpenAI API？"
   - 系统找：哪些文档和这个问题最相关？
   - 方法：计算问题向量和文档向量的相似度

2. **余弦相似度 = 比较向量的方向**
   - 方向一致 = 语义相似
   - 方向相反 = 语义相反
   - 只看方向，不看长度（文本长短不影响相似度）

3. **Embedding 模型的作用 = 把语义相似的文本转成方向相似的向量**
   - "Python 教程" 和 "Python 学习指南" → 方向相似
   - "Python 教程" 和 "天气预报" → 方向完全不同

   ```python
   results = vector_db.similarity_search(query, k=3)  # 自动返回最相似的 3 个
   ```

5. **向量数据库的作用**：在百万级的文档向量中，快速找出和问题向量方向最接近的那几个

---

## 总结

Embedding 是 RAG 的基础技术，核心要点：

1. **本质**：把文字转换成数字向量，让计算机能计算语义相似度
2. **用途**：语义搜索、文本聚类、去重、推荐
3. **原理**：语义相似的文本 → 方向相似的向量 → 余弦相似度高
4. **实现**：调用 API（OpenAI）或使用开源模型（BGE）
5. **关键**：选对模型、批量处理、缓存优化

掌握 Embedding 后，就能理解 RAG 的检索原理了。
