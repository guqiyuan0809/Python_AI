# 向量数据库（Vector Database）

## 什么是向量数据库？

向量数据库是专门用来存储和搜索向量的数据库，是 RAG 系统的核心组件。
不仅可以存储向量数据，也可以进行搜索

### 为什么需要向量数据库？

传统数据库（MySQL、PostgreSQL）：
- 擅长精确匹配：`WHERE name = 'Python'`
- 不擅长相似度搜索：找出"和 Python 相关的所有文档"

向量数据库：
- 专门为相似度搜索设计
- 能快速找出"最相似的 K 个向量"
- 支持海量向量的高效检索

### 核心功能

```python
# 1. 存入向量
db.add(
    text="Python 是一门编程语言",
    vector=[0.2, 0.8, 0.1, ...],
    metadata={"source": "doc1.pdf", "page": 1}
)

# 2. 相似度搜索
results = db.search(
    query_vector=[0.3, 0.7, 0.2, ...],
    top_k=3  # 返回最相似的 3 个
)

# 3. 过滤搜索
results = db.search(
    query_vector=[0.3, 0.7, 0.2, ...],
    filter={"source": "doc1.pdf"},  # 只在 doc1.pdf 中搜索
    top_k=3
)
```

## 常用向量数据库对比

### 1. Chroma（推荐学习使用）

**特点**：
- 轻量级，易上手
- 纯 Python 实现
- 支持本地文件存储
- 开箱即用，无需额外安装服务

**安装**：
```bash
pip install chromadb
```

**基本使用**：
```python
import chromadb

# 创建客户端
client = chromadb.Client()

# 创建集合（类似表）  向量数据库，集合=表
collection = client.create_collection(name="my_docs")

# 添加文档
collection.add(
    documents=["Python 是一门编程语言", "FastAPI 是 Web 框架"],
    ids=["doc1", "doc2"],
    metadatas=[{"source": "book1"}, {"source": "book2"}]
)

# 搜索
results = collection.query(
    query_texts=["如何学习编程？"],
    n_results=2
)

print(results)
```

**适用场景**：
- 学习和原型开发
- 小型项目（< 100 万条数据）
- 单机部署

### 2. Milvus（推荐生产使用）

**特点**：
- 生产级，性能强
- 支持分布式部署
- 支持 GPU 加速
- 功能丰富（混合搜索、分区、索引优化）

**安装**：
```bash
# 使用 Docker 启动 Milvus
docker-compose up -d

# 安装 Python 客户端
pip install pymilvus
```

**基本使用**：
```python
from pymilvus import connections, Collection, FieldSchema, CollectionSchema, DataType

# 连接 Milvus
connections.connect(host="localhost", port="19530")

# 定义集合结构
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=1536),
    FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=65535)
]
schema = CollectionSchema(fields=fields)
collection = Collection(name="my_docs", schema=schema)

# 插入数据
vectors = [[0.1, 0.2, ...], [0.3, 0.4, ...]]
texts = ["文档1", "文档2"]
collection.insert([vectors, texts])

# 创建索引
collection.create_index(field_name="vector", index_params={"index_type": "IVF_FLAT"})

# 搜索
collection.load()
results = collection.search(
    data=[[0.2, 0.3, ...]],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=3
)
```

**适用场景**：
- 生产环境
- 大规模数据（> 100 万条）
- 需要高性能和高可用

### 3. Pinecone（云服务）

**特点**：
- 完全托管的云服务
- 开箱即用，无需运维
- 按使用量付费
- 性能稳定

**安装**：
```bash
pip install pinecone-client
```

**基本使用**：
```python
import pinecone

# 初始化
pinecone.init(api_key="your-api-key", environment="us-west1-gcp")

# 创建索引
pinecone.create_index("my-index", dimension=1536, metric="cosine")

# 连接索引
index = pinecone.Index("my-index")

# 插入向量
index.upsert(vectors=[
    ("id1", [0.1, 0.2, ...], {"text": "文档1"}),
    ("id2", [0.3, 0.4, ...], {"text": "文档2"})
])

# 搜索
results = index.query(
    vector=[0.2, 0.3, ...],
    top_k=3,
    include_metadata=True
)
```

**适用场景**：
- 快速验证想法
- 不想自己运维
- 预算充足

### 4. Qdrant

**特点**：
- Rust 编写，性能优秀
- 支持丰富的过滤条件
- 支持多向量（一个文档多个向量）
- 有云服务和自部署两种方式

**安装**：
```bash
# Docker 启动
docker run -p 6333:6333 qdrant/qdrant

# Python 客户端
pip install qdrant-client
```

**基本使用**：
```python
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct

# 连接
client = QdrantClient(host="localhost", port=6333)

# 创建集合
client.create_collection(
    collection_name="my_docs",
    vectors_config=VectorParams(size=1536, distance=Distance.COSINE)
)

# 插入数据
client.upsert(
    collection_name="my_docs",
    points=[
        PointStruct(
            id=1,
            vector=[0.1, 0.2, ...],
            payload={"text": "文档1", "source": "book1"}
        )
    ]
)

# 搜索
results = client.search(
    collection_name="my_docs",
    query_vector=[0.2, 0.3, ...],
    limit=3
)
```

**适用场景**：
- 需要复杂过滤条件
- 性能要求高
- 生产环境

### 5. FAISS（Facebook AI）

**特点**：
- Facebook 开源
- 性能极强，支持 GPU
- 但只是一个库，不是完整的数据库
- 需要自己管理数据持久化

**安装**：
```bash
pip install faiss-cpu  # CPU 版本
# 或
pip install faiss-gpu  # GPU 版本
```

**基本使用**：
```python
import faiss
import numpy as np

# 创建索引
dimension = 1536
index = faiss.IndexFlatL2(dimension)  # L2 距离

# 添加向量
vectors = np.random.random((100, dimension)).astype('float32')
index.add(vectors)

# 搜索
query = np.random.random((1, dimension)).astype('float32')
distances, indices = index.search(query, k=3)

print(f"最相似的 3 个向量索引: {indices}")
print(f"距离: {distances}")
```

**适用场景**：
- 需要极致性能
- 愿意自己管理数据
- 研究和实验

## 向量数据库的核心概念 📖 了解即可

### 1. 距离度量（Distance Metric）

衡量两个向量相似度的方法：

| 度量 | 说明 | 范围 | 适用场景 |
|------|------|------|---------|
| 余弦相似度（Cosine）| 计算向量夹角 | -1 到 1（1 最相似）| 文本相似度 |
| 欧氏距离（L2）| 计算直线距离 | 0 到 ∞（0 最相似）| 图像、音频 |
| 点积（Dot Product）| 向量内积 | -∞ 到 ∞ | 推荐系统 |

**Chroma 默认使用 L2 距离**，但可以配置：

```python
collection = client.create_collection(
    name="my_docs",
    metadata={"hnsw:space": "cosine"}  # 使用余弦相似度
)
```

**为什么了解即可**：默认配置已经够用，不需要深入研究。

---

### 2. 索引类型 ⏭️ 跳过

向量数据库使用索引加速搜索：

| 索引类型 | 说明 | 优点 | 缺点 |
|---------|------|------|------|
| Flat | 暴力搜索，遍历所有向量 | 100% 准确 | 慢 |
| IVF | 聚类索引 | 快 | 略有精度损失 |
| HNSW | 图索引 | 快且准确 | 内存占用大 |

**Chroma 默认使用 HNSW**，性能和准确度平衡很好。

**为什么跳过**：这是底层实现细节，不影响使用。

## 实战：构建一个完整的向量数据库应用 ⏭️ 暂时跳过

```python
import chromadb
from chromadb.utils import embedding_functions
from typing import List, Dict

class VectorDB:
    def __init__(self, db_path="./chroma_db", collection_name="documents"):
        """初始化向量数据库"""
        self.client = chromadb.PersistentClient(path=db_path)
        
        # 使用 OpenAI Embedding
        self.embedding_function = embedding_functions.OpenAIEmbeddingFunction(
            api_key="your-api-key",
            model_name="text-embedding-3-small"
        )
        
        self.collection = self.client.get_or_create_collection(
            name=collection_name,
            embedding_function=self.embedding_function,
            metadata={"hnsw:space": "cosine"}
        )
    
    def add_documents(self, documents: List[str], metadatas: List[Dict] = None):
        """添加文档"""
        ids = [f"doc_{i}" for i in range(len(documents))]
        
        self.collection.add(
            documents=documents,
            ids=ids,
            metadatas=metadatas
        )
        
        print(f"成功添加 {len(documents)} 个文档")
    
    def search(self, query: str, top_k: int = 3, filter_dict: Dict = None):
        """搜索相关文档"""
        results = self.collection.query(
            query_texts=[query],
            n_results=top_k,
            where=filter_dict
        )
        
        return {
            'documents': results['documents'][0],
            'distances': results['distances'][0],
            'metadatas': results['metadatas'][0]
        }
    
    def delete_by_source(self, source: str):
        """删除特定来源的所有文档"""
        self.collection.delete(where={"source": source})
        print(f"已删除来源为 {source} 的所有文档")
    
    def get_stats(self):
        """获取数据库统计信息"""
        count = self.collection.count()
        return {"total_documents": count}

# 使用示例
if __name__ == "__main__":
    # 初始化
    db = VectorDB()
    
    # 添加文档
    documents = [
        "Python 是一门高级编程语言",
        "FastAPI 是一个现代化的 Web 框架",
        "机器学习是人工智能的一个分支"
    ]
    metadatas = [
        {"source": "book1", "category": "编程"},
        {"source": "book2", "category": "Web"},
        {"source": "book3", "category": "AI"}
    ]
    db.add_documents(documents, metadatas)
    
    # 搜索
    results = db.search("如何学习 Web 开发？", top_k=2)
    for doc, distance in zip(results['documents'], results['distances']):
        print(f"距离 {distance:.3f}: {doc}")
    
    # 过滤搜索
    results = db.search("编程", filter_dict={"category": "编程"})
    
    # 统计
    stats = db.get_stats()
    print(f"数据库中共有 {stats['total_documents']} 个文档")
```

**为什么暂时跳过**：这是封装好的工具类，等做完整项目时再用。现在先理解基础操作。

## 性能优化

### 1. 批量操作

```python
# ❌ 低效：逐个添加
for doc in documents:
    collection.add(documents=[doc], ids=[...])

# ✅ 高效：批量添加
collection.add(documents=documents, ids=[...])
```

```python
# 不要一次检索太多
results = collection.query(query_texts=["..."], n_results=5)  # 而不是 100
```

## 常见问题

### Q1：向量数据库和传统数据库有什么区别？

| 特性 | 传统数据库 | 向量数据库 |
|------|-----------|-----------|
| 查询方式 | 精确匹配 | 相似度搜索 |
| 索引 | B-Tree | HNSW/IVF |
| 适用场景 | 结构化数据 | 非结构化数据（文本、图像）|

### Q2：向量数据库能替代传统数据库吗？

**不能**。它们解决不同的问题：
- 传统数据库：存储结构化数据，精确查询
- 向量数据库：存储向量，相似度搜索

实际项目中通常两者结合使用。

### Q3：向量数据库的数据能持久化吗？

**能**。大部分向量数据库都支持持久化：
- Chroma：`PersistentClient`
- Milvus：自动持久化
- Qdrant：自动持久化

### Q4：向量数据库能存储多少数据？

| 数据库 | 规模 |
|--------|------|
| Chroma | 百万级 |
| Milvus | 十亿级 |
| Pinecone | 十亿级 |
| FAISS | 取决于内存 |
