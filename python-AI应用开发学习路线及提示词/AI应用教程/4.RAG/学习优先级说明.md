## 4.文档处理Pipeline.md

### 必练 ⭐
1. **文本分块（Text Chunking）**
   - 这是 RAG Pipeline 中最关键的一步
   - 必须理解重叠分块的原理
   - 手写一次 `chunk_by_length` 函数

2. **分块参数选择指南**
   - 理解 chunk_size 和 overlap 的作用
   - 记住推荐配置：chunk_size=500, overlap=50

### 了解即可 📖
1. **文档加载（Document Loading）**
   - LangChain 已经提供了现成的加载器
   - 知道有哪些格式支持即可

2. **使用 LangChain 的文档加载器**
   - 等做完整项目时会用到

3. **文本提取与清洗**
   - 理解概念，LangChain 会自动处理

4. **元数据管理**
   - 知道可以添加哪些元数据即可

5. **使用 LangChain 的文本分块器**
   - 等做完整项目时会用到

### 跳过 ⏭️
1. **手写 DocumentLoader 类**
   - 太复杂，直接用 LangChain

2. **手写 TextCleaner 类**
   - LangChain 会自动处理

3. **其他分块策略**（按段落、按句子、递归分块）
   - 先用固定长度重叠分块，够用了

4. **完整的 DocumentProcessor 类**
   - 封装好的工具类，等做完整项目时再用

5. **完整的 Pipeline 示例**
   - 等做完整项目时再看

6. **高级技巧**（智能分块、保留文档结构、多粒度分块）
   - 优化技巧，先跑通基础 RAG

7. **常见问题 Q3-Q4**（表格图片、代码文件）
   - 遇到再查

## 学习建议

### 第一步：理解核心概念（1 小时）
- 快速浏览三个文件，理解 RAG 的三大核心
- 不需要记住所有细节

### 第二步：动手练习必练内容（2 小时）
1. Embedding 调用 + 相似度计算（30 分钟）
2. Chroma 基本操作（30 分钟）
3. 文本分块实践（1 小时）

### 第三步：直接进入 RAG 完整流程（重点）
- 不要在单个模块上花太多时间
- 用 LangChain 做完整的 RAG 系统
- 在实践中理解各个模块的作用

### 核心原则
1. **理论 > 示例代码**：理解原理比会写代码更重要
2. **最小验证即可**：跑通一个最简单的示例就够了
3. **快速进入实战**：重点是做出完整的 RAG 系统
4. **遇到问题再回来查**：文档是参考资料，不是教科书
