# Agent Skills 核心概念

> Agent Skills 是让 AI 从"只会聊天"变成"能干活"的核心能力集合

**逐层递进关系：**

```
Agent Skill（单个能力）
  = 一个工具函数 + Function Calling

Agent Skills（多个能力）
  = 多个工具 + 记忆 + 流式输出

Agent（完整智能体）
  = Agent Skills + 循环决策 + 规划
```

每一层都是在上一层基础上加东西，不是推翻重来。

---

## 什么是 Agent Skills

Agent Skills 是构建智能体（Agent）的基础能力，让 AI 不仅能理解和生成文本，还能：
- 调用外部工具和 API
- 记住对话历史
- 拆解和规划复杂任务
- 实时流式返回结果

---

## 普通 AI vs 有 Agent Skills 的 AI

### 普通 AI（只会聊天）
```
用户：北京天气怎么样？
AI：抱歉，我无法实时查询天气信息，我的知识截止到 2023 年...
```

### 有 Agent Skills 的 AI
```
用户：北京天气怎么样？
AI：[自动调用天气 API] 
    北京今天晴，气温 25°C，空气质量良好，适合户外活动。
```

**关键区别：** 有 Agent Skills 的 AI 能主动调用工具获取实时信息，而不是只依赖训练数据。

## Agent Skills 的四大核心能力

### 1. 工具调用（Tool Use / Function Calling）

**定义：** 让 AI 能够调用外部函数、API 和工具。

**应用场景：**
- 查询天气 API
- 查询数据库
- 执行 Python 代码
- 发送邮件
- 搜索网页
- 调用计算器

**工作原理：**
```python
# 1. 你定义一个工具函数
def get_weather(city: str) -> str:
    """查询指定城市的天气"""
    # 实际调用天气 API
    return f"{city}今天晴，25°C"

# 2. 把函数注册给 AI
# 3. AI 自动判断何时调用
用户输入："北京天气怎么样？"
AI 思考：这需要实时天气信息 → 调用 get_weather("北京") → 返回结果
```

**示例：会查天气的 AI**
```python
def get_weather(city: str) -> str:
    """查询城市天气"""
    return f"{city}今天晴，25°C"

def query_database(sql: str) -> str:
    """执行 SQL 查询"""
    # 连接数据库执行查询
    pass

# AI 会根据用户问题自动选择调用哪个工具
```

**这就是一个 Agent Skill！** 让 AI 具备了"查天气"的能力。

### 2. 记忆（Memory）

**定义：** 让 AI 记住对话历史和上下文。

**应用场景：**
- 多轮对话
- 记住用户偏好
- 保持对话连贯性

**示例：**
```
用户：我叫张三
AI：你好张三，很高兴认识你！

用户：我叫什么名字？
AI：你叫张三

用户：我喜欢吃什么？
AI：抱歉，你还没告诉我你喜欢吃什么
```

**实现方式：**
- 短期记忆：保存当前对话的所有消息
- 长期记忆：将重要信息存入数据库或向量数据库
- 工作记忆：只保留最近 N 轮对话

**长期记忆用什么存储？**

| 存储内容 | 推荐方式 | 原因 |
|---------|---------|------|
| 用户偏好（名字、喜好、设置） | MySQL/PostgreSQL/JSON文件 | 结构化数据，精确查询 |
| 历史对话记录 | MySQL/PostgreSQL | 按时间、用户 ID 查询 |
| 需要语义检索的记忆 | 向量数据库（Chroma） | 按相似度找相关记忆 |

大部分场景用传统数据库就够了。比如记住"用户叫张三、喜欢 Python"，存 MySQL 一条记录就行。

只有需要"模糊回忆"时才用向量数据库。比如用户问"我之前问过关于部署的问题"，需要从几百条历史对话里按语义找出相关的那几条，这时候才需要向量检索（原理和 RAG 一样：Embedding + 向量数据库 + 相似度搜索）。

实际项目中最常见的组合：MySQL 存结构化的用户信息和对话记录，需要语义检索时再加一个向量数据库。

### 3. 规划（Planning）

**定义：** 让 AI 能够拆解复杂任务，制定执行计划。

**应用场景：**
- 数据分析任务
- 多步骤操作
- 复杂问题求解

**示例：**
```
用户：帮我分析这个 Excel 文件的销售数据

AI 规划：
  步骤 1：读取 Excel 文件
  步骤 2：清洗数据（去除空值、异常值）
  步骤 3：计算统计指标（总销售额、平均值、增长率）
  步骤 4：生成可视化图表
  步骤 5：输出分析报告

然后 AI 会按照这个计划一步步执行
```

**常见规划模式：**
- ReAct（推理 + 行动）：边思考边执行
- Plan-and-Execute：先制定完整计划，再执行
- Tree of Thoughts：探索多个可能的解决路径

**规划是怎么实现的？**

规划不是什么特殊技术，本质就是通过 prompt 让 AI 自己想步骤，然后配合 Function Calling 循环一步步执行。

**ReAct 模式（边想边做）：** AI 不需要提前规划好所有步骤，而是每一步都思考"我现在该做什么"，做完看结果，再想下一步。

**循环是怎么实现的？** 就是你的代码用 while 循环反复调用 AI API：

```python
while True:
    # 调用 AI（带上工具和之前的对话历史）
    response = client.chat.completions.create(model=MODEL, messages=messages, tools=tools)
    
    if response 要调用工具:
        # 执行工具，把结果加入 messages
        # 继续循环，再次调用 AI
    else:
        # AI 不再调用工具，给出最终回答
        break  # 退出循环
```

每一轮循环就是一次 API 调用。AI 看到完整的 messages（包括之前的工具调用结果），自己决定：
- 还需要调用工具 → 继续循环
- 信息够了 → 给出最终回答，循环结束

**实际执行过程：**
```
第 1 轮 API 调用：AI 看到用户问题，决定调用 read_excel → 你执行，结果加入 messages
第 2 轮 API 调用：AI 看到文件数据，决定调用 clean_data → 你执行，结果加入 messages
第 3 轮 API 调用：AI 看到清洗后的数据，决定调用 calculate → 你执行，结果加入 messages
第 4 轮 API 调用：AI 看到计算结果，觉得够了 → 生成最终分析报告，循环结束
```

所以"规划"能力来自 LLM 本身的推理能力，"循环"是你代码里的 while 循环反复调用 API。两者结合就是 Agent。

### 4. 流式输出（Streaming）

**定义：** 实时逐字返回 AI 的回复，而不是等全部生成完再返回。

**用户体验对比：**

**非流式输出：**
```
用户：介绍一下 Python
[等待 5 秒...]
AI：Python 是一种高级编程语言，由 Guido van Rossum 于 1991 年创建...
```

**流式输出：**
```
用户：介绍一下 Python
AI：Python█
AI：Python 是█
AI：Python 是一种█
AI：Python 是一种高级█
AI：Python 是一种高级编程█
AI：Python 是一种高级编程语言█...
```

**优势：**
- 用户体验更好，感觉 AI 在"思考"
- 减少等待焦虑
- 可以提前看到部分结果

## Agent Skills 与 LangChain 的关系

### LangChain 是什么

LangChain 是一个 **AI 应用开发框架**，专门用于构建基于大语言模型的应用。

**类比：**
- 大语言模型（GPT-4、Claude）= 汽车引擎
- LangChain = 汽车制造工具包
- Agent Skills = 汽车的各种功能（导航、倒车雷达、自动驾驶）

### LangChain 提供了什么

LangChain 封装了实现 Agent Skills 所需的所有工具和组件：

1. **工具调用（Tools）**
   - 提供了大量预定义的工具（搜索、计算器、数据库查询等）
   - 简化了自定义工具的创建
   - 自动处理工具调用的逻辑

2. **记忆（Memory）**
   - ConversationBufferMemory：保存所有对话
   - ConversationSummaryMemory：总结对话历史
   - VectorStoreMemory：基于向量数据库的长期记忆

3. **规划（Agents）**
   - ReAct Agent：推理 + 行动模式
   - Plan-and-Execute Agent：先规划再执行
   - 自动处理任务拆解和执行流程

4. **流式输出（Streaming）**
   - 内置流式输出支持
   - 简化了流式 API 的实现

### 为什么用 LangChain 学习 Agent Skills

**不用 LangChain（手写所有逻辑）：**
```python
# 需要自己实现工具调用、记忆管理、规划逻辑
# 代码量大，容易出错
def my_agent():
    # 100+ 行代码实现工具调用
    # 50+ 行代码实现记忆管理
    # 80+ 行代码实现规划逻辑
    pass
```

**用 LangChain：**
```python
from langchain.agents import create_react_agent
from langchain.tools import Tool

# 定义工具
weather_tool = Tool(
    name="get_weather",
    func=get_weather,
    description="查询城市天气"
)

# 创建 Agent（自动具备工具调用、记忆、规划能力）
agent = create_react_agent(llm, tools=[weather_tool])
agent.run("北京天气怎么样？")
```

**LangChain 的价值：**
- 大幅减少代码量
- 提供最佳实践
- 快速原型开发
- 专注业务逻辑，而不是底层实现

## Agent Skills 在学习路线中的位置

```
1. Skills（基础技能）
   ↓ 学会调用 AI API
   
2. Ollama（本地模型）
   ↓ 学会部署和使用本地模型
   
3. MCP（连接外部工具）
   ↓ 学会让 AI 连接外部数据和工具
   
4. RAG（知识库问答）★核心
   ↓ 学会让 AI 基于文档回答问题
   
5. Agent Skills（智能体核心能力）★核心  ← 你在这里
   ↓ 学会让 AI 调用工具、记忆、规划
   
6. Agent（构建完整智能体）★核心
   ↓ 把所有能力组合起来，构建完整的智能体
   
7. 多 Agent 协作
   ↓ 多个智能体协同工作
```

**关系：**
- RAG：让 AI 能基于你的文档回答问题（知识增强）
- Agent Skills：让 AI 能调用工具、执行任务（能力增强）
- Agent：把 RAG + Agent Skills 组合起来，构建完整的智能体
