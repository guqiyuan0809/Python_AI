# 记忆机制详解

> 你在 Skills 阶段已经实现过多轮对话（维护 messages 列表），那就是最基础的记忆。
> 这里讲 Agent 场景下更完整的记忆体系。

---

## 一、为什么 Agent 需要记忆

普通聊天：记住上下文就行。

Agent 干活时需要记住更多东西：
- 用户之前说了什么（对话历史）
- 工具调用的中间结果（"我刚查了天气是 25°C"）
- 用户的偏好（"这个用户喜欢简洁回答"）
- 长期知识（"上次用户问过退货政策"）

---

## 二、三种记忆类型

### 1. 短期记忆（对话历史）

就是你已经会的 messages 列表：

```python
messages = [
    {"role": "system", "content": "你是一个助手"},
    {"role": "user", "content": "我叫张三"},
    {"role": "assistant", "content": "你好张三"},
    {"role": "user", "content": "我叫什么？"},
]
# AI 能回答"你叫张三"，因为历史消息里有
```

问题：messages 越来越长，Token 越来越多，成本越来越高，最终会超出上下文窗口。

### 2. 工作记忆（滑动窗口）

只保留最近 N 轮对话，老的丢掉：

```python
MAX_HISTORY = 10  # 只保留最近 10 轮

def add_message(messages, role, content):
    messages.append({"role": role, "content": content})
    # 保留 system prompt + 最近 N 轮
    if len(messages) > MAX_HISTORY * 2 + 1:
        # 保留第一条 system prompt
        messages[:] = [messages[0]] + messages[-(MAX_HISTORY * 2):]
    return messages
```

优点：控制 Token 成本。
缺点：早期的对话信息丢失了。

### 3. 长期记忆（持久化存储）

把重要信息存到数据库或文件里，需要时再取出来：

```python
# 存储用户偏好
user_memory = {
    "name": "张三",
    "preference": "喜欢简洁回答",
    "last_topic": "Python Web 开发"
}

# 下次对话时，把相关记忆加入 system prompt
system_prompt = f"""你是一个助手。
用户信息：{user_memory}
请根据用户偏好回答。"""
```

或者用向量数据库存储（和 RAG 一样的原理）：
- 把历史对话向量化存入 Chroma
- 新对话时，检索相关的历史记录
- 加入上下文

## 三、Agent 中的记忆实现

Agent 执行多步任务时，需要记住中间结果：

```
用户：帮我查北京天气，如果超过 30 度就提醒我带伞

第 1 步：调用 get_weather("北京") → 结果：35°C，晴
         记忆：北京 35°C

第 2 步：AI 看到记忆中 35°C > 30°C
         生成提醒：温度超过 30 度，建议做好防晒

这里 AI 需要"记住"第 1 步的结果，才能在第 2 步做判断
```

在代码层面，这就是把 tool 调用的结果放进 messages 列表：

```python
messages = [
    {"role": "user", "content": "查北京天气，超过30度提醒我"},
    {"role": "assistant", "tool_calls": [...]},           # AI 要调用工具
    {"role": "tool", "content": "35°C，晴"},              # 工具结果
    # AI 看到完整的 messages，知道温度是 35°C，可以做判断
]
```

---

## 四、实际开发中的记忆策略

| 场景 | 记忆方式 | 实现 |
|------|---------|------|
| 简单聊天 | 短期记忆（messages 列表） | 你已经会了 |
| 控制成本 | 工作记忆（滑动窗口） | 只保留最近 N 轮 |
| 记住用户偏好 | 长期记忆（数据库） | 存到 MySQL/文件 |
| 检索历史对话 | 向量记忆（向量数据库） | 存到 Chroma，按相似度检索 |

---

## 总结

- 记忆的本质就是管理 messages 列表 + 持久化存储
- 短期记忆你已经会了（Skills 阶段的多轮对话）
- Agent 场景多了工具调用的中间结果需要记住
- 长期记忆就是存数据库，需要时取出来塞进 prompt
- 不需要单独花太多时间，做 Agent 项目时自然会用到
