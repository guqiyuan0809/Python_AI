# Function Calling 详解

> Function Calling 是 Agent Skills 的核心中的核心。
> 理解了它，就理解了 Agent 是怎么"干活"的。

---

## 一、Function Calling 是什么

让 AI 自己决定什么时候调用你写的函数。

之前你调 API 的流程是：
```
用户提问 → AI 生成文字回答 → 结束
```

有了 Function Calling：
```
用户提问 → AI 判断"我需要调用某个函数" → 返回函数名和参数 → 你执行函数 → 把结果给 AI → AI 生成最终回答
```

关键点：AI 不会自己执行函数，它只是告诉你"我想调哪个函数、传什么参数"，执行是你的代码做的。

换句话说：AI 是"决策者"，你的代码是"执行者"。AI 负责判断该调哪个函数、传什么参数，你的程序负责真正执行这个函数，拿到结果后再发回给 AI，AI 再基于结果生成最终回答。整个过程中 AI 从头到尾没有直接执行过任何函数。

---

## 二、工作流程（6 步）

```
第 1 步：你定义工具函数（比如查天气、查数据库）
第 2 步：你把函数的描述告诉 AI（函数名、参数、用途）
第 3 步：用户提问
第 4 步：AI 判断是否需要调用函数
         - 不需要 → 直接回答
         - 需要 → 返回要调用的函数名和参数
第 5 步：你的代码执行函数，拿到结果
第 6 步：把函数执行结果发给 AI，AI 生成最终回答
```

用具体例子：
```
用户："北京今天天气怎么样？"

AI 思考：这个问题需要实时天气数据，我自己不知道，
        但我有一个 get_weather 工具可以用。

AI 返回：请调用 get_weather，参数 city="北京"

你的代码：执行 get_weather("北京")，得到 "晴天，25°C"

你把结果发给 AI：工具返回了 "晴天，25°C"

AI 最终回答：北京今天晴天，气温 25°C，适合出门。
```

## 三、tools 参数结构

在调用 API 时，通过 `tools` 参数告诉 AI 有哪些函数可用：

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",           # 函数名
            "description": "查询指定城市的天气",  # 函数描述（AI 靠这个判断什么时候用）
            "parameters": {                   # 参数定义（JSON Schema 格式）
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称，如北京、上海"
                    }
                },
                "required": ["city"]
            }
        }
    }
]
```

三个关键字段：
- `name`：函数名，AI 返回时会带上这个名字
- `description`：函数描述，AI 靠这个判断什么时候该调用这个函数（写清楚很重要）
- `parameters`：参数定义，告诉 AI 这个函数需要什么参数、什么类型

## 四、AI 的返回结构

当 AI 决定调用函数时，返回的不是文字，而是一个 tool_calls 对象：

```python
# 普通回答（不需要调用函数）
response.choices[0].message.content = "Python 是一门编程语言..."
response.choices[0].message.tool_calls = None

# 需要调用函数时
response.choices[0].message.content = None  # 没有文字内容
response.choices[0].message.tool_calls = [
    {
        "id": "call_abc123",
        "type": "function",
        "function": {
            "name": "get_weather",          # 要调用的函数名
            "arguments": '{"city": "北京"}'  # 参数（JSON 字符串）
        }
    }
]
```

你的代码需要：
1. 检查 `tool_calls` 是否存在
2. 解析函数名和参数
3. 执行对应的函数
4. 把结果发回给 AI

## 五、完整的消息流程

Function Calling 涉及多轮消息传递：

```python
messages = [
    # 第 1 轮：用户提问
    {"role": "user", "content": "北京天气怎么样？"},
    
    # 第 2 轮：AI 说"我要调用函数"（不是文字回答）
    {"role": "assistant", "content": None, "tool_calls": [...]},
    
    # 第 3 轮：你把函数执行结果告诉 AI
    {"role": "tool", "tool_call_id": "call_abc123", "content": "晴天，25°C"},
    
    # 第 4 轮：AI 基于函数结果生成最终回答
    # （这一轮是 AI 的响应，不在 messages 里，而是 response）
]
```

注意第 3 轮的 role 是 `"tool"`，不是 `"user"` 也不是 `"assistant"`，这是专门用来传递工具执行结果的。

## 六、AI 怎么决定调不调用函数

AI 根据两个东西判断：
1. 用户的问题内容
2. 你提供的工具描述（description）

```
用户问："北京天气怎么样？"
AI 看到有 get_weather 工具，描述是"查询指定城市的天气"
→ 匹配！调用 get_weather

用户问："Python 是什么？"
AI 看到 get_weather 工具，但这个问题和天气无关
→ 不调用，直接用自己的知识回答
```

所以 `description` 写得好不好，直接影响 AI 的判断准确度。

从这里来说，agent智能体，不仅是有具体功能的AI，它也可以基于自己的训练数据进行回答，
更加强大

## 七、多个工具

可以同时注册多个工具，AI 自己选：

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "查询指定城市的实时天气",
            "parameters": {...}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "calculator",
            "description": "计算数学表达式，如加减乘除",
            "parameters": {...}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "search_web",
            "description": "搜索互联网获取最新信息",
            "parameters": {...}
        }
    }
]

# 用户问"北京天气" → AI 选 get_weather
# 用户问"123 * 456" → AI 选 calculator
# 用户问"最新的 iPhone 价格" → AI 选 search_web
# 用户问"Python 是什么" → AI 不调用任何工具，直接回答
```

## 八、Function Calling 和 MCP 的关系

你在 MCP 阶段学过，MCP 也是让 AI 调用工具。区别是：

| | Function Calling | MCP |
|--|--|--|
| 用在哪 | 你自己写的 AI 后端代码里 | AI IDE（Cursor/Kiro）等应用里 |
| 谁来执行函数 | 你的代码 | MCP Server |
| 工具定义在哪 | API 请求的 tools 参数里 | 独立的 MCP Server 进程里 |

Function Calling 是你做 Agent 项目时用的，MCP 是给现成 AI 应用加功能用的。底层思路一样，实现方式不同。

## 九、和 RAG 的关系

RAG 其实可以作为 Agent 的一个工具：

```python
tools = [
    {
        "name": "search_knowledge_base",
        "description": "从知识库中搜索相关文档",
        ...
    },
    {
        "name": "get_weather",
        "description": "查询天气",
        ...
    }
]

# 用户问"公司年假政策" → AI 调用 search_knowledge_base（RAG 检索）
# 用户问"北京天气" → AI 调用 get_weather
```

Agent = Function Calling + RAG + 记忆 + 规划，全部组合起来。

---

## 总结

- Function Calling 的本质：AI 告诉你"我想调哪个函数"，你执行后把结果给它
- AI 不执行函数，只做决策
- 通过 tools 参数注册工具，通过 description 让 AI 理解工具用途
- 返回的 tool_calls 包含函数名和参数，你解析后执行
- 这是 Agent 的核心能力，后面学 Agent 就是在这个基础上加循环和规划
