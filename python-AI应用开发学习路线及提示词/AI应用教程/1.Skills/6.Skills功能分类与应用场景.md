# Skills 功能分类与应用场景

> Skills 的本质：用 API 调用大模型，通过不同的参数组合实现不同的功能。
调用API大模型，通过不同的参数组合，实现不同的功能

## 零、补充理解：Skill 不只是 Prompt

上面的说法抓住了大方向，但还需要补一句：

**Skills 不只是 Prompt，Prompt 是核心之一，但不是全部。**

更准确地说，Skill 本质上像一个**可复用的能力包装**，通常会把这几类东西组合在一起：

### 1. 系统指令 / Prompt

定义这个能力要怎么思考、怎么输出、遵守什么规则。这通常确实是最核心的部分。

### 2. 工具使用规则

比如什么时候该查文档、什么时候该改文件、什么时候该跑命令。也就是说，Skill 往往不仅告诉模型“说什么”，还告诉它“怎么做”。

这里要注意区分：
- 真正执行工具调用的是 **Agent**
- Skill 本身更像是告诉 Agent：在什么情况下用什么工具、按什么方式用

### 3. 工作流（先干什么，再干什么，这就是工作流，是提示词的一种）

例如先读 `SKILL.md`，再找模板，再调用脚本，再生成结果。这个流程本身也是能力的一部分。

### 4. 上下文资源

包括模板、示例、参考文档、脚本、素材等。没有这些，很多 Skill 就只是一个普通提示词；有了这些，才更像一个完整功能模块。

---

### 可以这样记

```text
大模型 = 通用推理引擎
Prompt = 行为和角色设定
Skill = Prompt + 工具规则 + 流程 + 资源 的组合封装
Agent = 真正执行任务、并调用工具的主体
Tool = Agent 可调用的外部能力
```

换句话说：

- 如果只改 Prompt，你是在“调模型的说话和思考方式”
- 如果做成 Skill，你是在“封装一个稳定可复用的能力”

所以说“核心参数就是 Prompt 提示词”，这个判断**大方向是对的**；但如果把 Skill 完全等同于“换一套提示词”，就有点低估它了。很多真正好用的 Skill，强的不只是提示词，而是**提示词把工具、流程和资源组织起来的能力**。

## 一、按功能类型划分

| 功能类型 | 核心技术 | 说明 |
|---------|---------|------|
| 文本生成 | 基础调用 | 生成文章、摘要、翻译、改写 |
| 角色扮演 | System Prompt | 让 AI 扮演特定角色（客服、导师、面试官） |
| 对话交互 | 多轮对话 + 历史记录 | 聊天机器人、连续问答 |
| 实时响应 | 流式输出（Stream + SSE） | 面向用户的聊天界面 |
| 代码辅助 | System Prompt + 低 temperature | 代码生成、代码审查、Bug 修复 |
| 数据处理 | 基础调用 + 格式约束 | 提取信息、分类、格式转换 |
| 内容审核 | System Prompt + 规则约束 | 判断内容是否合规、敏感词检测 |

## 二、根据网站需求编写指定功能

不同类型的网站，需要的 AI 功能不同，但底层都是 API 调用 + 参数组合：

### 电商网站
```python
# 智能客服：回答商品问题、处理售后
messages = [
    {"role": "system", "content": "你是XX商城的客服，只回答商品和订单相关问题，不回答无关内容。"},
    {"role": "user", "content": "这个手机支持5G吗？"}
]

# 商品描述生成：根据参数自动生成卖点文案
messages = [
    {"role": "system", "content": "你是电商文案专家，根据商品参数生成吸引人的描述，100字以内。"},
    {"role": "user", "content": "商品：无线蓝牙耳机，续航30小时，主动降噪，重量5g"}
]
```

### 教育网站
```python
# AI 导师：根据学生水平讲解知识
messages = [
    {"role": "system", "content": "你是一个编程导师，学生是Python初学者，用简单的语言和类比解释概念。"},
    {"role": "user", "content": "什么是装饰器？"}
]

# 自动出题：根据知识点生成练习题
messages = [
    {"role": "system", "content": "你是出题老师，根据知识点出3道选择题，包含答案和解析。输出JSON格式。"},
    {"role": "user", "content": "知识点：Python列表推导式"}
]
```

### 内容平台
```python
# 文章摘要：自动生成文章摘要
messages = [
    {"role": "system", "content": "用一段话总结以下文章的核心内容，不超过100字。"},
    {"role": "user", "content": "（文章全文）"}
]

# 内容审核：判断用户发布的内容是否合规
messages = [
    {"role": "system", "content": "判断以下内容是否包含违规信息（广告、色情、暴力、政治敏感）。只回答JSON：{\"safe\": true/false, \"reason\": \"原因\"}"},
    {"role": "user", "content": "（用户发布的内容）"}
]
```

### 企业内部系统
```python
# 数据分析助手：用自然语言查询数据
messages = [
    {"role": "system", "content": "你是SQL专家，根据用户的自然语言描述生成SQL查询语句。数据库表：users(id,name,age,city), orders(id,user_id,amount,date)"},
    {"role": "user", "content": "查一下上个月北京用户的总消费金额"}
]

# 会议纪要生成
messages = [
    {"role": "system", "content": "根据会议记录，提取：1.讨论要点 2.决策事项 3.待办任务及负责人。用Markdown格式输出。"},
    {"role": "user", "content": "（会议录音转文字内容）"}
]
```

## 三、常用场景及实现方式

| 场景 | 实现方式 | 关键参数 |
|------|---------|---------|
| 智能客服 | System Prompt 限定角色 + 多轮对话 + 流式输出 | temperature=0.3，限定回答范围 |
| 文案生成 | System Prompt 设定风格 + 基础调用 | temperature=0.7-1.0，更有创意 |
| 代码生成 | System Prompt 设定语言和规范 + 基础调用 | temperature=0-0.3，更准确 |
| 翻译 | 基础调用 + 指定源语言和目标语言 | temperature=0，确定性翻译 |
| 数据提取 | System Prompt 指定输出 JSON 格式 + 基础调用 | temperature=0，格式稳定 |
| 内容审核 | System Prompt 定义规则 + 返回 JSON 判断 | temperature=0，结果一致 |
| 文章摘要 | System Prompt 限定字数和风格 + 基础调用 | temperature=0.3 |
| 对话机器人 | System Prompt + 多轮对话 + 流式输出 + 错误处理 | 全套组合 |

## 四、实现模式总结

所有功能的底层都是同一套代码，区别只在于参数组合：

```python
def ai_function(user_input: str, system_prompt: str, temp: float = 0.7, max_tokens: int = 500):
    """通用 AI 功能函数，改参数就能实现不同功能"""
    response = client.chat.completions.create(
        model=MODEL,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_input}
        ],
        temperature=temp,
        max_tokens=max_tokens
    )
    return response.choices[0].message.content

# 客服
ai_function("退货流程是什么", "你是XX商城客服...", temp=0.3)

# 文案
ai_function("蓝牙耳机，续航30h", "你是文案专家...", temp=0.9)

# 翻译
ai_function("Hello World", "将英文翻译成中文，只输出翻译结果", temp=0)

# 代码
ai_function("写一个快速排序", "你是Python工程师，直接给代码", temp=0.1)
```

**核心思路：一个通用函数 + 不同的 System Prompt + 不同的 temperature = 无限种功能。**

## 五、进阶：Skills 的局限

Skills 能做的是"单次智能"，有些事情它做不到：

| 做不到的事 | 需要的技术 | 学习阶段 |
|-----------|-----------|---------|
| 基于公司文档回答问题 | RAG（检索增强生成） | 第 4 阶段 |
| 自动调用 API、查数据库 | Agent（工具调用） | 第 5-6 阶段 |
| 记住用户偏好和历史 | 持久化记忆（数据库） | 第 5 阶段 |
| 处理超长文档 | 文档分块 + RAG | 第 4 阶段 |
| 多步骤自动完成复杂任务 | Agent（规划 + 执行） | 第 6 阶段 |

**Skills 是基础，所有功能底层都是同一套 API 调用代码，区别只在于 System Prompt + temperature + 参数组合。后面学 RAG 和 Agent，就是在这个基础上叠加更强的能力。**

**写好 Skills 的三个关键：**

1. **模型选对** — 不同模型擅长的领域不同，成本也不同，根据场景选合适的模型
2. **参数合理** — temperature、max_tokens 这些要根据场景调，写代码用低温，创意用高温
3. **Prompt 写好** — System Prompt 越清晰、约束越明确，输出质量越高

**模型和参数决定了能力上限，Prompt 决定了实际发挥。**

## 六、技巧：用 AI 写 Prompt

不需要自己从零写 System Prompt，让 AI 帮你写，效率更高、质量更好。

**工作流：**
```
1. 你把业务需求描述清楚（用户是谁、流程是什么、边界在哪、什么能做什么不能做）
2. 让 AI 生成初版 Prompt
3. 用初版 Prompt 测试效果
4. 不满意再让 AI 优化
5. 反复迭代直到满意
```

**示例：**
```
你跟 AI 说：
"帮我写一个电商客服的 System Prompt，要求：
- 只回答商品和订单相关问题
- 语气友好专业
- 遇到无法解决的问题引导用户联系人工客服
- 不回答与业务无关的问题"

AI 会生成一个比你自己写的更完整、更规范的 Prompt。
```

**关键前提：你必须把业务讲明白。** AI 不懂你的业务，你得告诉它用户是谁、业务流程是什么、边界条件有哪些。你负责想清楚要什么，AI 负责写出来。
