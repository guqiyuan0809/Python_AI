# AI API 调用

> API 调用是 AI 应用开发的基础，后面的 RAG、Agent 全都建立在这之上。

---

## 调用流程

```
1. 获取 API Key（去模型平台注册）
2. 安装 SDK：pip install openai
3. 创建 client，指定 base_url 和 api_key
4. 构造 messages 消息列表
5. 调用 client.chat.completions.create()
6. 处理返回结果
```

---

## 基础代码

```python
from openai import OpenAI

client = OpenAI(
    api_key="你的key",
    base_url="https://api.deepseek.com"  # 换模型只改这里
)

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "你是一个 Python 工程师"},
        {"role": "user", "content": "什么是 FastAPI？"}
    ]
)

print(response.choices[0].message.content)
```

---

## messages 三种角色

| 角色 | 作用 | 是否必须 |
|------|------|---------|
| system | 设定 AI 的身份、行为、约束 | 可选，但推荐 |
| user | 用户的输入 | 必须 |
| assistant | AI 的历史回答（实现上下文记忆） | 多轮对话时需要 |

---

## 返回结果

```python
# 获取回答文本
response.choices[0].message.content

# choices 是一个列表（一般只有一个元素）
# [0] 取第一个回答
# .message.content 就是文本内容

# Token 用量
response.usage.prompt_tokens       # 输入 Token
response.usage.completion_tokens   # 输出 Token
response.usage.total_tokens        # 总 Token
```

---

## temperature 参数

控制回答的随机性，范围 0-2：

| 值 | 效果 | 适用场景 |
|----|------|---------|
| 0 | 每次回答几乎一样，最确定 | 需要稳定输出的场景 |
| 0 - 0.3 | 比较稳定 | 写代码、技术问答 |
| 0.5 - 0.7 | 平衡创造性和准确性 | 日常对话、客服 |
| 0.7 - 1.0 | 更随机、更有创意 | 创意写作、文案、头脑风暴 |

**默认值是 0.7，写代码建议设 0-0.3。**

## 5 种调用方式及应用场景

### 1. 基础调用
一问一答，不需要上下文。
```python
response = client.chat.completions.create(
    model=MODEL,
    messages=[{"role": "user", "content": "翻译：Hello World"}],
    max_tokens=200
)
```
**场景**：翻译、解释概念、格式转换、单次问答

### 2. 带 System Prompt（角色设定）
给 AI 设定身份和行为规则。
```python
messages=[
    {"role": "system", "content": "你是 Python 后端工程师，回答简洁，直接给代码"},
    {"role": "user", "content": "FastAPI 怎么做 JWT？"}
]
```
**场景**：特定领域助手、客服机器人、限定回答范围

### 3. 多轮对话（上下文记忆）
把历史对话全部带上，AI 才能理解上下文。
```python
messages=[
    {"role": "user", "content": "FastAPI 怎么做参数校验？"},
    {"role": "assistant", "content": "用 Pydantic..."},   # AI 上轮的回答
    {"role": "user", "content": "给个代码示例"}            # 追问
]
```
**场景**：聊天应用、连续追问、技术讨论

**注意**：AI 本身不记忆，"上下文"是你在代码里维护 messages 列表实现的。程序退出后历史就丢了，要持久化需要存到文件或数据库。

### 4. 流式输出（Stream）
AI 边生成边返回，用户能看到文字逐渐出现。
```python
response = client.chat.completions.create(
    model=MODEL,
    messages=[{"role": "user", "content": "写一篇文章"}],
    stream=True  # 开启流式
)
for chunk in response:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```
**场景**：任何面向用户的聊天界面、RAG 知识库问答、长回答场景

### 5. 带错误处理 + Token 统计
生产环境必须有的防护措施。
```python
try:
    response = client.chat.completions.create(
        model=MODEL,
        messages=[...],
        max_tokens=1000
    )
    answer = response.choices[0].message.content
    tokens = response.usage.total_tokens
except Exception as e:
    # 处理异常：Key 过期、余额不足、网络超时等
    print(f"出错了: {e}")
```
**场景**：任何上线的 AI 应用，用来监控成本、处理异常、保证稳定性

---

## 实际项目中的组合

5 种方式不是单独用的，而是组合起来：

```
完整的聊天应用 = System Prompt + 多轮对话 + 流式输出 + 错误处理

RAG 知识库问答 = System Prompt + 检索到的文档作为上下文 + 流式输出 + 错误处理

Agent 智能体 = System Prompt + 多轮对话 + 工具调用 + 流式输出 + 错误处理
```

## 换模型只改两行

```python
# DeepSeek
client = OpenAI(api_key="sk-xxx", base_url="https://api.deepseek.com")

# OpenRouter（一个 Key 调多种模型）
client = OpenAI(api_key="sk-or-xxx", base_url="https://openrouter.ai/api/v1")

# 本地 Ollama（免费）
client = OpenAI(api_key="ollama", base_url="http://localhost:11434/v1")

# OpenAI 官方
client = OpenAI(api_key="sk-xxx")
```

**代码里做好抽象，不要把 Key 和 URL 写死，方便随时切换模型。**
