# 主流大模型对比

> 了解各模型的特点，开发时知道选哪个。

---

## 模型对比

| 模型 | 厂商 | 优势 | 劣势 | 适用场景 |
|------|------|------|------|---------|
| GPT-4o | OpenAI | 综合能力强，生态最成熟 | 贵，国内需要代理 | 对质量要求高的场景 |
| Claude 3.5/4 | Anthropic | 长文本处理强，代码能力好 | 国内需要代理 | 代码生成、长文档分析 |
| DeepSeek V3/R1 | 深度求索 | 性价比极高，推理能力强，国内直连 | 高峰期可能排队 | 日常开发、成本敏感场景 |
| 通义千问 Qwen | 阿里 | 中文能力好，国内直连，有免费额度 | 英文场景稍弱 | 中文场景、阿里云生态 |
| 文心一言 ERNIE | 百度 | 中文理解好，国内直连 | API 生态不如 OpenAI | 中文场景、百度云生态 |
| Llama 3 | Meta | 开源免费，可本地部署 | 需要自己部署，中文一般 | 本地部署、隐私敏感场景 |

---

## 怎么选

- **学习和开发阶段**：用 DeepSeek，便宜够用，国内直连
- **对质量要求高**：用 GPT-4o 或 Claude
- **成本敏感的生产环境**：DeepSeek 或通义千问
- **数据不能出本地**：用 Ollama 跑 Llama/Qwen 开源模型
- **做海外产品**：OpenAI 或 Claude，生态最成熟

---

## 实际开发中的建议

1. 开发调试用便宜模型（DeepSeek），上线前用好模型测试效果
2. 大部分 API 都兼容 OpenAI 格式，换模型只需要改 base_url 和 api_key
3. 不要绑死一个模型，代码里做好抽象，方便随时切换
